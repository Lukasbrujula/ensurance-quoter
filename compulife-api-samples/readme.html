1. Introduction

The Compulife API has been designed to give access to the COMPULIFE Internet Quote Engine software through the use of JSON formatted data. The API has sections that are public (anyone can use) or private (must have a COMPULIFE Authorization ID). Only one server is allowed to access the COMPULIFE API per Authorization ID, and this is tracked by matching the ID with the IP address. The first time you use your COMPULIFEAUTHORIZATIONID your IP used for that request is logged and is assumed to be the one you will be using. Moving your code to a new server will require you to contact us to reset your information. This applies to development/live server moves as well, if your development server and live server have a different IP address.


2. Overview

The API can be used directly, or you can use a local php/javascript library which does the heavy lifting of the interface for you.

API functions include:

  • State/Province lists that can be used.
  • Company logos that can be used in your results.
  • List of life companies that are available to quote, for the United States or Canada.
  • Lists of life companies with their available products to quote.
  • Running quotes (including only the companies and products that you want want to quote).
  • Running spreadsheet styled results.
  • Running quotes with our health analyzer questionnaire.

4. Sample Requests

Similar to our Postman Collection, api-sample-requests.php references samples that can be used from a local computer or a server. It does not require any php code in order to run, however the results returned are in JSON format, so this serves as reference example only. Most browsers can add a plugin or extension to format JSON into something more readable, but this is obviously is not suitable for anything other then development reference purposes.

Some sample requests do not require an Authorization ID. If an Authorization ID is required, you will need to configure your config.php page.

As you go through the examples, the bottom of the page shows the url that is generated in order to get results from the API.

IMPORTANT: Your COMPULIFE Authorization ID is only valid for one IP address, so if you are doing development on a different server you will have to call COMPULIFE to reset your Compulife Authorization ID.


5. Sample Requests That Require Authorization ID

Quoter-sample.php shows sample quote forms and will show working results. It uses the local COMPULIFE API php and javascript code to call the engine and will handle logging of calls. In order to use this config.php must be updated with

  1. Authorization ID
    • i.e. - ['COMPULIFEAUTHORIZATION'] = 'INSERT_ID'
  2. Server path to where you want your log files to be recorded
    • i.e. - ['log_path'] = '/home/yourserverpath/compulifeapi.logs/'

You also need to create the directory of where your log files will be put.

There are three examples of lookups with quoter-sample.php.

  1. Quote Results - shows the results in ajax style on the same page.
  2. Side by Side Spreadsheet - again shows the results in ajax style on the same page.
  3. Health Analyzer - continues to the health analyzer questionnaire page which will ask more detailed questions about the client's health. This page is fairly complicated, so we suggest limiting your editing to light css type of editing. Clicking submit on this page goes to results.php, which does the actual API lookup and will show the results in ajax style on the same page.

IMPORTANT: Your COMPULIFE Authorization ID is only valid for one IP address, so if you are doing development on a different server you will have to call us to reset your Compulife Authorization ID.


6. Sample Log file and debugging tips

Log file names are formatted as log-YEAR-MM-DD.log
The detail can be set in the config.php, the default is '4' which is show all.

A sample log entry is:
INFO - 2020-07-11 00:08:50-->===== Request Start =====
INFO - 2020-07-11 00:08:50-->json={"COMPULIFEAUTHORIZATIONID":"INSERT_ID","BirthDay":"20","BirthMonth":"9","BirthYear":"1987","CompRating":"4","ErrOnMissingZipCode":"ON","FaceAmount":"250000","Health":"PP","ModeUsed":"M","NewCategory":"7","Sex":"M","Smoker":"N","SortOverride1":"A","State":"0","ZipCode":"84081"}
INFO - 2020-07-11 00:08:50-->api request string = https://www.compulifeapi.com/api/request/?COMPULIFE={"COMPULIFEAUTHORIZATIONID":"INSERT_ID","BirthDay":"20","BirthMonth":"9","BirthYear":"1987","CompRating":"4","ErrOnMissingZipCode":"ON","FaceAmount":"250000","Health":"PP","ModeUsed":"M","NewCategory":"7","Sex":"M","Smoker":"N","SortOverride1":"A","State":"0","ZipCode":"84081"}

The API request string can put put into the address bar of your browser, and will show results in JSON. This is very useful in debugging where things go wrong. Keep in mind that the COMPULIFEAUTHORIZATIONID for your IP address must match, and if your browser is at a different IP address then the server you are debugging that will affect your results. In this case, make a tiny html page on your server that has the request string as a link, in order to get results.
