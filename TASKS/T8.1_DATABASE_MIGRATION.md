# T8.1: Database — ai_agents Table + ai_transcripts Table

**Priority:** Critical — everything in Phase 8 depends on this
**Estimate:** 1 hr
**Phase:** 8 (Agent Management + Transcripts + Usage)
**Depends on:** Phase 7 complete

---

## Context

Phase 7 built a single-agent system: one Telnyx AI assistant per Ensurance agent, with the assistant ID stored on `agent_settings`. Phase 8 expands this to support multiple AI agents per user, each with its own configuration, phone number, and call history. We also need proper transcript storage for AI conversations (message-level, not flat text).

## 9 Pillars

### 1. Objective
Create `ai_agents` table for multi-agent support, `ai_transcripts` table for message-level conversation storage, and migrate existing Phase 7 data to the new schema.

### 2. Scope
- New `ai_agents` table
- New `ai_transcripts` table
- Add `ai_agent_id` FK to existing `ai_agent_calls` table
- Migrate existing single-agent data from `agent_settings` to `ai_agents`
- RLS on all new tables

### 3. Dependencies
- Supabase project access (SQL editor or MCP)
- Existing tables: `agent_settings` (has `ai_agent_id`, `ai_agent_enabled`), `ai_agent_calls` (from T7.1)
- Check actual schema before writing migration: `SELECT column_name, data_type FROM information_schema.columns WHERE table_name = 'ai_agent_calls'`

### 4. Implementation

**ai_agents table:**
```sql
CREATE TABLE IF NOT EXISTS ai_agents (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  agent_id UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
  telnyx_assistant_id TEXT,
  name TEXT NOT NULL DEFAULT 'Insurance Intake Agent',
  description TEXT,
  status TEXT NOT NULL DEFAULT 'inactive',  -- 'active', 'inactive', 'error'
  phone_number TEXT,                         -- E.164 format, the number this agent answers
  greeting TEXT,                             -- What the AI says when it picks up
  system_prompt TEXT,                        -- Full system prompt (generated or custom)
  voice TEXT DEFAULT 'Telnyx.NaturalHD.astra',
  model TEXT DEFAULT 'Qwen/Qwen3-235B-A22B',
  total_calls INTEGER DEFAULT 0,
  total_minutes NUMERIC(10,2) DEFAULT 0,
  last_call_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

ALTER TABLE ai_agents ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Users can manage their own agents"
  ON ai_agents FOR ALL
  USING (agent_id = auth.uid())
  WITH CHECK (agent_id = auth.uid());

CREATE INDEX idx_ai_agents_agent_id ON ai_agents(agent_id);
```

**ai_transcripts table:**
```sql
CREATE TABLE IF NOT EXISTS ai_transcripts (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  call_id UUID NOT NULL,                    -- FK to ai_agent_calls
  ai_agent_id UUID NOT NULL REFERENCES ai_agents(id) ON DELETE CASCADE,
  agent_id UUID NOT NULL REFERENCES auth.users(id),
  role TEXT NOT NULL,                        -- 'assistant', 'user', 'system'
  content TEXT NOT NULL,
  message_index INTEGER NOT NULL DEFAULT 0,  -- Order within conversation
  timestamp TIMESTAMPTZ,                     -- When this message was spoken
  created_at TIMESTAMPTZ DEFAULT NOW()
);

ALTER TABLE ai_transcripts ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Users can view their own transcripts"
  ON ai_transcripts FOR SELECT
  USING (agent_id = auth.uid());

CREATE POLICY "Users can insert their own transcripts"
  ON ai_transcripts FOR INSERT
  WITH CHECK (agent_id = auth.uid());

CREATE INDEX idx_ai_transcripts_call_id ON ai_transcripts(call_id, message_index);
CREATE INDEX idx_ai_transcripts_agent ON ai_transcripts(ai_agent_id, created_at DESC);
```

**Update ai_agent_calls table:**
```sql
-- Add FK to ai_agents (nullable for backward compat with Phase 7 calls)
ALTER TABLE ai_agent_calls ADD COLUMN IF NOT EXISTS ai_agent_id UUID REFERENCES ai_agents(id);
CREATE INDEX IF NOT EXISTS idx_ai_agent_calls_ai_agent ON ai_agent_calls(ai_agent_id, created_at DESC);
```

**Migration of existing data:**
```sql
-- For each agent that has an ai_agent_id in agent_settings, create an ai_agents row
INSERT INTO ai_agents (agent_id, telnyx_assistant_id, name, status)
SELECT user_id, ai_agent_id, 'Insurance Intake Agent', 
  CASE WHEN ai_agent_enabled = true THEN 'active' ELSE 'inactive' END
FROM agent_settings
WHERE ai_agent_id IS NOT NULL
ON CONFLICT DO NOTHING;

-- Link existing ai_agent_calls to the new ai_agents rows
UPDATE ai_agent_calls 
SET ai_agent_id = ai_agents.id
FROM ai_agents
WHERE ai_agent_calls.agent_id = ai_agents.agent_id
  AND ai_agent_calls.ai_agent_id IS NULL;
```

### 5. Success Criteria
- [ ] `ai_agents` table exists with all columns and RLS
- [ ] `ai_transcripts` table exists with all columns and RLS
- [ ] `ai_agent_calls` has `ai_agent_id` FK column
- [ ] Existing Phase 7 agent data migrated to `ai_agents` table
- [ ] Existing `ai_agent_calls` linked to corresponding `ai_agents` row
- [ ] RLS works: users can only see their own agents, transcripts, and calls
- [ ] Can insert a new ai_agent, create an ai_agent_call linked to it, and store transcript messages

### 6. Failure Handling
- If `ai_agent_calls` table doesn't exist (Phase 7 didn't create it), skip the FK addition and migration. The table will be created as part of the full schema.
- If `agent_settings` doesn't have `ai_agent_id` column, skip the data migration. No existing data to move.
- All ALTER TABLE uses IF NOT EXISTS / IF EXISTS to be idempotent.

### 7. Guardrails
- Do NOT drop any existing tables or columns
- Do NOT modify RLS on existing tables
- Do NOT modify application code in this task (that's T8.2+)
- All new columns nullable except where defaulted
- Migration must be idempotent (safe to run multiple times)

### 8. Human Checkpoint
- Run the migration SQL in Supabase SQL editor (or via MCP)
- Verify tables exist and RLS is active
- Verify data migration moved existing agent data correctly

### 9. Estimated Time
1 hour (including verification)
