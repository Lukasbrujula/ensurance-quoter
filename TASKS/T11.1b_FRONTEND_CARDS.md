### 1. Objective

Build the card components and redesign Call Mode in the AI assistant panel to display coaching cards instead of raw transcript. The transcript continues running in the background and gets saved — it just stops being the primary visual.

### 4. Implementation

#### A. Card Components

All cards use shadcn/ui `Card` as base with a colored left border (`border-l-4`).

**`components/coaching/style-card.tsx`**
- Border: `border-l-blue-500`
- Layout:
  ```
  🎯 COMMUNICATION STYLE
  
  Steady (S) · ●●●○
  Relationship-first, deliberate decision-maker
  
  → Match her pace. Don't rush.
  → She'll want to discuss with family.
  → Frame around protecting her family.
  ```
- Props: `StyleCard` from coaching types
- Confidence shown as filled/empty dots (1-4)
- Tips as a compact bulleted list with → prefix
- No dismiss button (style card stays)
- Never auto-collapses

**`components/coaching/medication-card.tsx`**
- Border: `border-l-amber-500`
- Layout:
  ```
  💊 MEDICATION DETECTED
  
  "Metformin" · Type 2 Diabetes
  
  ✅ JH: Preferred DB
  ✅ Foresters: Accept
  ❌ AMAM: Decline
  ❌ MOO: Decline
  
  ⚠️ Narrows options. JH strongest for diabetes.
  ```
- Props: `MedicationCard` from coaching types
- Carrier results as a compact grid: emoji (✅/❌/⚠️) + carrier name + detail
- Sort: accepts first, then conditionals, then declines
- Severity indicator: high severity gets a subtle amber background tint
- Dismiss × button
- Auto-collapses after 60 seconds to: `💊 Metformin → Type 2 Diabetes`

**`components/coaching/life-event-card.tsx`**
- Border: `border-l-green-500`
- Layout:
  ```
  👶 LIFE EVENT: New Grandchild
  
  Cross-sell opportunity:
  · Term life for the new parent
  · Children's rider on existing policy
  
  💡 TRY SAYING:
  "A lot of new parents don't think about 
  coverage until later. I could help your 
  daughter too if she's interested."
  ```
- Props: `LifeEventCard` from coaching types
- Cross-sell suggestions as bullet list
- Script in a slightly different style — lighter weight or italic, with quotes
- Dismiss × button
- Auto-collapses after 60 seconds to: `👶 New Grandchild — cross-sell opportunity`

**`components/coaching/coaching-tip-card.tsx`**
- Border: `border-l-gray-400`
- Layout:
  ```
  💡 SUGGESTED QUESTION
  
  Client mentioned her daughter.

  "Does your daughter have any

  ```
- Props: `CoachingTipCard` from coaching types
- Script (if present) in quotes, lighter weight
- Dismiss × button
- Auto-collapses after 30 seconds to: `💡 Suggested Question`

#### B. Card Stack (`components/coaching/coaching-card-stack.tsx`)

Container component that manages the card array:

```typescript
interface CoachingCardStackProps {
  cards: CoachingCard[];
  onDismiss: (cardId: string) => void;
}
```

**Behavior:**
- Style card always pinned at top (if present)
- Other cards in reverse chronological order (newest first below style card)
- New cards appear with a subtle CSS transition (opacity + translateY)
- Auto-collapse timers:
  - Style: never
  - Medication: 60 seconds after appearance
  - Life event: 60 seconds after appearance
  - Coaching tip: 30 seconds after appearance
- Collapsed state: single line showing emoji + short summary, click to expand
- Track collapsed state locally with `useState<Set<string>>`
- Max ~5 expanded cards visible, additional cards start collapsed
- Scroll area if cards exceed panel height
- When no cards exist, show empty state

**Empty state (before any cards appear):**
```
🎧 Listening...
Intelligence cards will appear as the 
conversation progresses.
```

#### C. Panel Redesign (`ai-assistant-panel.tsx`)

**Call Mode changes:**
- Replace transcript + coaching hints display with `<CoachingCardStack />`
- Read `coachingCards` from call-store (new field, see Part D)
- At the bottom of the panel, add a collapsible "Show Transcript" section:
  - Small text button/link: "📝 Show transcript"
  - Click toggles a scrollable area showing the raw transcript (existing data from call-store)
  - Default: collapsed (hidden)
  - This is for agents who want to see what was said — it's secondary

**Chat Mode: NO CHANGES.** Leave completely untouched. Only the Call Mode rendering changes.

**Transition between modes:**
- When a call starts → show Call Mode with card stack
- When call ends → cards remain visible for review (they get saved with the call log in T11.1c)
- When agent switches to Chat Mode manually → Chat Mode as-is

#### D. Call Store Updates (`stores/call-store.ts`)

Add to the store:

```typescript
// New state
coachingCards: CoachingCard[];

// New actions
addCoachingCard: (card: CoachingCard) => void;
// Appends card. If card.type === 'style' and a style card already exists, replace it.

// Remove card by ID
```

Keep existing `coachingHints` field for backward compatibility. New code uses `coachingCards`. Old hints still work until the coaching API caller is updated in T11.1c.

### 6. Failure Handling

- If `coachingCards` is empty → show "🎧 Listening..." empty state (not a broken UI)
- If a card has an unrecognized type → skip rendering it (don't crash the stack)
- If auto-collapse timer fires after card is dismissed → no-op (check card still exists)
- If Call Mode is active but call store has no call state → show empty state gracefully

### 7. Guardrails

- Do NOT modify Chat Mode — only Call Mode in the panel
- Do NOT modify the coaching API — that's T11.1a (already done)
- Do NOT wire the coaching API calls to the card stack — that's T11.1c
- Do NOT modify Telnyx/Deepgram/call lifecycle code
- Do NOT add new npm dependencies — shadcn Card + existing Tailwind + CSS transitions
- Cards must pass the "glance test" — readable in 2 seconds mid-conversation
- Do NOT make cards too wide — they live in the right panel which is typically 300-400px
