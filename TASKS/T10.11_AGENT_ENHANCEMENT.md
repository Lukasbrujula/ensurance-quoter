## 9 Pillars

### 1. Objective

The current agent create dialog is too bare — just name, description, phone, voice. Not enough to build a reliable agent. Three problems to fix:

1. **No structured prompt builder** — agents type a free-text description instead of configuring structured behavior
2. **No calendar integration** — AI-booked callbacks don't land on Google Calendar
3. **Post-login redirect goes to `/dashboard`** — should go to `/quote`

**Reference architecture:** The Growthly WhatsApp/Voice project has a battle-tested agent builder. Claude Code MUST study the Growthly files first, then build an insurance-specific simplified version.

**Part B — Prompt builder UI + compiler (1.5 hrs):**

**Part C — Post-call wiring (45 min):**

### 4. Implementation

#### Part A: Post-login redirect

One-line change. Find the redirect after login and change `/dashboard` to `/quote`.

#### Part B: Prompt Builder

**What to learn from Growthly → what to build for Ensurance:**

| Growthly Pattern | Ensurance Adaptation |
|---|---|
| Full NestJS backend with separate `agent_scenarios`, `scenario_fields` tables | Simplified: JSONB columns on `ai_agents` table |
| Multi-step wizard (7 steps) | 2 steps: template select → configure |
| `prompt-compiler.service.ts` with `compileVoice()` | New `lib/telnyx/prompt-compiler.ts` with `compileAgentPrompt()` |
| `locked-sections.ts` per industry | Insurance-specific locked section (never discuss rates/medical) |
| `healthcare.template.ts` with scenarios + fields | Update existing insurance templates with greeting, personality, collect_fields |
| Separate scenario CRUD | `collect_fields` JSONB array on ai_agents |

**New file: `lib/telnyx/prompt-compiler.ts`**

Pure function that takes agent config → returns compiled Telnyx prompt. Sections:

1. **LOCKED: Insurance Compliance** (user can't edit)
   - AI disclosure at call start
   - Never discuss rates, premiums, policy details, medical info
   - If caller objects to AI → collect callback number → end politely
   
2. **IDENTITY** — from `agent.personality`

3. **GREETING** — from `agent.greeting`

4. **DATA COLLECTION** — from `agent.collect_fields` (checkbox selections)
   - Maps field IDs to natural language prompts
   - "Ask ONE piece of information at a time"
   
5. **FAQ** — from `agent.faq_entries` (T10.8)

6. **BUSINESS HOURS** — from `agent.business_hours` (T10.8)

7. **CLOSING** — standard confirmation + handoff

8. **LOCKED: Voice Optimization** (user can't edit)
   - Max 3 sentences per response
   - Natural fillers, pronunciation rules
   - From Growthly's voice constraints pattern

#### Part C: Post-call wiring

In `ai-lead-processor.ts`, after call data extraction:

```typescript
const agent = await getAIAgent(agentId);
const actions = agent.post_call_actions || ['save_lead', 'book_calendar', 'send_notification'];

// 1. CRM Lead (existing, now respects toggle)
let lead = null;
if (actions.includes('save_lead')) {
  lead = await createOrUpdateLead(agentId, extractedData);
}

// 2. Google Calendar (non-blocking)
if (actions.includes('book_calendar') && extractedData.callback_time && lead) {
  createCalendarEvent(agent.owner_id, {
    title: `AI Callback: ${extractedData.name || 'Unknown caller'}`,
    description: `Booked by ${agent.name}. Reason: ${extractedData.reason || 'Not specified'}`,
    startTime: new Date(extractedData.callback_time),
    leadId: lead.id,
  }).then(eventId => {
    if (eventId) updateLead(lead.id, agent.owner_id, { google_event_id: eventId });
  }).catch(err => console.error('[Calendar] Failed:', err));
}

// 3. Notification via activity log
if (actions.includes('send_notification')) {
  await insertActivityLog({
    agent_id: agent.owner_id,
    lead_id: lead?.id,
    type: 'ai_callback_booked',
    details: { agent_name: agent.name, caller_name: extractedData.name, callback_time: extractedData.callback_time },
  });
}
```

- [ ] `npx tsc --noEmit` passes

### 6. Failure Handling

- Google Calendar not connected → `book_calendar` silently skips
- callback_time not collected → calendar event skipped, log warning
- Activity log insert fails → don't crash webhook handler
- All post-call actions are fire-and-forget (webhook responds 200 immediately)
- Existing agents without new columns get defaults

### 7. Guardrails

- **Do NOT modify any Telnyx WebRTC, webhook registration, or SIP code** — Telnyx is not forgiving
- Only change: (a) what prompt text gets sent, (b) what happens after call ends
- Do NOT expose raw compiled prompt to the agent UI
- Do NOT let agents remove the locked insurance compliance section
- Do NOT break existing agents — new columns have sensible defaults
- Prompt compiler is a pure function — no side effects, easy to test
