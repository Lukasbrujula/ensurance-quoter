# T9.6: Webhook Signature Verification

**Priority:** Medium — prevents forged webhook payloads from creating fake leads
**Estimate:** 45 min
**Phase:** 9 (Security Hardening)

---

## 9 Pillars

### 1. Objective
Verify that incoming webhook requests actually originate from Telnyx, not from an attacker who discovered the webhook URL. Replace the current URL-based secret token approach with proper cryptographic signature verification.

### 2. Scope
- Telnyx webhook signature verification on all Telnyx-originating endpoints
- Validate the webhook signing secret from Telnyx dashboard
- Reject requests with missing or invalid signatures
- Keep the URL secret token as a secondary layer (belt and suspenders)

### 3. Dependencies
- Current webhook endpoints. Find them:
  ```bash
  find app/api -path "*webhook*" -o -path "*telnyx*" | grep route.ts
  grep -rn "webhook\|INTERNAL_API_SECRET" --include="*.ts" app/api/
  ```
- Telnyx webhook signing documentation: Telnyx signs webhooks with an ED25519 signature. Check their docs for the exact verification method.
- Telnyx dashboard: the webhook signing public key is available under API Keys or Webhook Settings

### 4. Implementation

**How Telnyx webhook signing works:**

Telnyx includes these headers on webhook requests:
- `telnyx-signature-ed25519` — The ED25519 signature of the payload
- `telnyx-timestamp` — Unix timestamp of when the signature was generated

The verification process:
1. Get the raw request body (as string, not parsed JSON)
2. Concatenate: `timestamp|payload`
3. Verify the ED25519 signature against Telnyx's public key

**Create verification utility:**

```typescript
// lib/middleware/telnyx-webhook-verify.ts

import crypto from 'crypto';

// Telnyx's public signing key — from Telnyx dashboard or their docs
// This is a public key, safe to commit (it's not a secret)
const TELNYX_PUBLIC_KEY = process.env.TELNYX_WEBHOOK_PUBLIC_KEY;

// Maximum age of a webhook signature (5 minutes)
const MAX_TIMESTAMP_AGE_MS = 5 * 60 * 1000;

export function verifyTelnyxWebhook(
  rawBody: string,
  signature: string | null,
  timestamp: string | null
): { valid: boolean; reason?: string } {
  // If no public key configured, skip verification (dev mode)
  if (!TELNYX_PUBLIC_KEY) {
    if (process.env.NODE_ENV === 'production') {
      console.error('[Webhook] TELNYX_WEBHOOK_PUBLIC_KEY not set in production!');
      return { valid: false, reason: 'Webhook verification not configured' };
    }
    console.warn('[Webhook] Skipping signature verification (no public key configured)');
    return { valid: true };
  }

  if (!signature || !timestamp) {
    return { valid: false, reason: 'Missing signature or timestamp headers' };
  }

  // Check timestamp freshness (prevent replay attacks)
  const webhookTime = parseInt(timestamp, 10) * 1000; // Convert to ms
  const now = Date.now();
  if (Math.abs(now - webhookTime) > MAX_TIMESTAMP_AGE_MS) {
    return { valid: false, reason: 'Webhook timestamp too old (possible replay attack)' };
  }

  // Verify ED25519 signature
  try {
    const signedPayload = `${timestamp}|${rawBody}`;
    const signatureBuffer = Buffer.from(signature, 'base64');
    const publicKeyBuffer = Buffer.from(TELNYX_PUBLIC_KEY, 'base64');

    const isValid = crypto.verify(
      null, // ED25519 doesn't use a separate hash algorithm
      Buffer.from(signedPayload),
      {
        key: publicKeyBuffer,
        format: 'der',
        type: 'spki',
      },
      signatureBuffer
    );

    if (!isValid) {
      return { valid: false, reason: 'Invalid signature' };
    }

    return { valid: true };
  } catch (error) {
    console.error('[Webhook] Signature verification error:', error);
    return { valid: false, reason: 'Signature verification failed' };
  }
}
```

**Note on ED25519 verification:** The exact verification method depends on how Telnyx formats their public key. Before implementing, check:
1. Telnyx webhook signing docs for the exact algorithm and key format
2. Whether they provide an SDK or npm package for verification (`telnyx` npm package may have a `Telnyx.Webhook.constructEvent()` helper)
3. Test with a real webhook to confirm the signature format

If Telnyx provides an official verification method, use that instead of rolling our own.

**Apply to webhook endpoints:**

```typescript
// In the webhook route handler:
export async function POST(request: Request) {
  // Get raw body BEFORE parsing as JSON
  const rawBody = await request.text();
  
  const signature = request.headers.get('telnyx-signature-ed25519');
  const timestamp = request.headers.get('telnyx-timestamp');

  const verification = verifyTelnyxWebhook(rawBody, signature, timestamp);
  if (!verification.valid) {
    console.warn('[Webhook] Rejected:', verification.reason);
    return Response.json({ error: 'Invalid webhook signature' }, { status: 401 });
  }

  // Now parse the verified body
  const body = JSON.parse(rawBody);
  // ... rest of handler
}
```

**Important: raw body access**

The signature is computed over the raw body string. If you parse JSON first and re-stringify, the signature won't match. You MUST:
1. Read the body as text first (`request.text()`)
2. Verify the signature against the raw text
3. Parse JSON after verification

**Environment variable:**
```
TELNYX_WEBHOOK_PUBLIC_KEY=<base64 encoded public key from Telnyx dashboard>
```

**Keep the URL secret as backup:**

Don't remove the existing `?token=INTERNAL_API_SECRET` check. Keep both:
```typescript
// Layer 1: Telnyx signature verification
const sigValid = verifyTelnyxWebhook(rawBody, sig, ts);

// Layer 2: URL token (existing, keep as fallback)
const tokenValid = url.searchParams.get('token') === process.env.INTERNAL_API_SECRET;

// In production: require signature. Token is backup for dev/testing.
// In development: either is fine.
if (!sigValid.valid && !tokenValid) {
  return Response.json({ error: 'Unauthorized' }, { status: 401 });
}
```

### 5. Success Criteria
- [ ] Webhook verification utility created
- [ ] All Telnyx webhook endpoints verify signatures before processing
- [ ] Raw body is preserved for signature verification (not pre-parsed)
- [ ] Stale timestamps (>5 min old) are rejected (replay protection)
- [ ] Missing signatures are rejected in production
- [ ] Development mode allows unverified webhooks (for local testing without ngrok signatures)
- [ ] Existing URL token check is preserved as secondary layer
- [ ] Valid Telnyx webhooks still process correctly
- [ ] `npx tsc --noEmit` passes

### 6. Failure Handling
- If `TELNYX_WEBHOOK_PUBLIC_KEY` is not set in production, log a loud error and reject webhooks (don't silently accept unverified requests)
- If `TELNYX_WEBHOOK_PUBLIC_KEY` is not set in development, skip verification with a warning (allow local testing)
- If the ED25519 verification library behaves differently than expected, fall back to the URL token check and log the issue
- If Telnyx changes their signing format, the URL token still works as backup

### 7. Guardrails
- Do NOT parse the request body before verifying the signature
- Do NOT remove the existing URL token authentication (keep as backup)
- Do NOT hardcode the public key (use environment variable)
- Do NOT block webhooks in development when the public key isn't configured
- Do NOT assume the exact signature format without testing against a real Telnyx webhook
- If Telnyx provides an official SDK method for verification, prefer that over custom crypto code

### 8. Human Checkpoint
- Get the webhook signing public key from Telnyx dashboard
- Set it as `TELNYX_WEBHOOK_PUBLIC_KEY` env var
- Trigger a real AI agent call and verify the webhook still processes
- Send a forged POST to the webhook URL without Telnyx headers — should get 401
