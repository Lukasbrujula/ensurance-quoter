# T8.3: /agents List Page — Agent Cards + Create Flow

**Priority:** High — the primary UI for agent management
**Estimate:** 1.5 hrs
**Phase:** 8 (Agent Management + Transcripts + Usage)
**Depends on:** T8.2 (API routes exist)

---

## Context

Currently AI agent config lives in Settings > Integrations (Phase 7). We're moving it to a dedicated `/agents` page that supports multiple agents. This is a top-level nav item alongside Leads, Quotes, and Settings.

## 9 Pillars

### 1. Objective
Build `/agents` page with agent list (cards), create agent dialog, status management, and navigation integration.

### 2. Scope
- New route: `app/agents/page.tsx`
- Agent card component
- Create agent dialog/sheet
- Navigation: add "Agents" to top nav
- Protect route with auth middleware
- Settings > Integrations: replace AI agent panel with link to /agents

### 3. Dependencies
- T8.2 completed (API routes for CRUD)
- Existing navigation component (find with: `grep -rn "TopNav\|Navigation\|nav" --include="*.tsx" components/navigation/`)
- Existing middleware route protection (find: `middleware.ts` in project root)
- Existing card/dialog patterns (Settings page, Leads list as reference)
- shadcn/ui: Card, Button, Dialog, Input, Select, Badge, Switch, Skeleton (all installed)

### 4. Implementation

**Page layout:**
```
/agents
+-------------------------------------------------+
| Agents                        [+ Create Agent]  |
+-------------------------------------------------+
| [Agent Card 1]  [Agent Card 2]  [Agent Card 3]  |
|                                                  |
| [Empty state if no agents]                       |
+-------------------------------------------------+
```

**Agent card design:**
Each agent displays as a Card with:
- Agent name (bold, top)
- Status badge: green "Active" / gray "Inactive" / red "Error"
- Phone number (if assigned)
- Stats row: "X calls | Y minutes"
- Last call: "2 hours ago" or "No calls yet"
- Quick actions: Enable/Disable toggle, Edit button, link to detail page

**Card grid:**
- Desktop: 3 columns (`grid-cols-1 md:grid-cols-2 lg:grid-cols-3`)
- Mobile: single column, full width cards
- Cards link to `/agents/[id]` on click (except toggle and edit actions)

**Create agent dialog:**
- Triggered by "+ Create Agent" button
- Fields:
  - Name (required) — default: "Insurance Intake Agent"
  - Description (optional) — what this agent handles
  - Phone number (optional) — dropdown of available Telnyx numbers or text input for E.164
  - Voice (optional) — select from available voices, default: "Telnyx.NaturalHD.astra"
- On submit:
  - Call `POST /api/agents`
  - Show loading state on button
  - On success: close dialog, refresh list, show toast "Agent created"
  - On error: show error in dialog, keep form populated

**Empty state:**
When agent has no AI agents yet:
- Illustration or icon (Robot/Bot icon from Lucide)
- "No AI agents yet"
- "Create your first AI voice agent to handle calls when you're unavailable"
- Large "Create Agent" CTA button

**Navigation update:**
- Add "Agents" to the top nav bar, between "Leads" (or "Quotes") and "Settings"
- Icon: Bot or Phone (from Lucide)
- Active state when on /agents or /agents/[id]

**Route protection:**
- Add `/agents/*` to the protected routes in middleware.ts (same pattern as /leads, /quote, /settings)

**Settings > Integrations update:**
- Replace the AI Agent configuration panel from Phase 7 with a simpler card:
  - "AI Voice Agents" card with count of active agents
  - "Manage Agents" button linking to /agents
  - Keep other Coming Soon integration cards as-is

### 5. Success Criteria
- [ ] `/agents` page renders with agent cards or empty state
- [ ] "Agents" appears in top navigation with correct active state
- [ ] Create agent dialog opens, validates input, calls API
- [ ] New agent appears in list after creation (no page reload needed)
- [ ] Agent cards show name, status badge, phone number, call stats
- [ ] Enable/disable toggle works directly from the card
- [ ] Cards link to `/agents/[id]` detail page
- [ ] Empty state renders when no agents exist
- [ ] Route is protected (redirects to login if not authenticated)
- [ ] Settings > Integrations links to /agents instead of inline config
- [ ] Responsive: 3-col desktop, 2-col tablet, 1-col mobile
- [ ] `npx tsc --noEmit` passes

### 6. Failure Handling
- If API returns error on list fetch, show error state with "Retry" button
- If create fails, keep dialog open with error message (don't lose form data)
- If toggle fails, revert the switch to previous state and show error toast
- If an agent has status 'error', show a "Retry Setup" action on the card

### 7. Guardrails
- Do NOT remove the Settings > Integrations page (other integrations will live there)
- Do NOT modify the existing quote or leads pages
- Do NOT create a Zustand store for agents unless the existing patterns use one for similar data (check if leads/settings use a store or just SWR/fetch)
- Follow existing nav patterns exactly (don't introduce a new nav component)
- Follow existing card patterns from the codebase

### 8. Human Checkpoint
- Visual: does the agents page look consistent with the rest of the app?
- Navigation: does "Agents" appear correctly and highlight when active?
- Create flow: can you create an agent and see it appear?
- Responsive: does the grid collapse properly on mobile?
