# T7.3: Inbound Call Webhook — Receive AI Conversation Data

**Priority:** High — this is how AI call data gets into our system  
**Estimate:** 1 hr  
**Phase:** 7 (Telnyx AI Agent)  
**Depends on:** T7.1 (AI service wrapper)

---

## Context

When the Telnyx AI assistant finishes a call, it can send structured data to a webhook. The webhook tool we configured in T7.2 sends the extracted caller info (name, reason, callback time, etc.) to our endpoint. We also need to handle Telnyx's post-call conversation events to get the full transcript and metadata.

There are two data flows:
1. **Webhook tool call** — Telnyx AI calls our webhook mid-conversation or at end with structured data extracted by the AI
2. **Conversation completion** — We can poll Telnyx for conversation details + transcript after the call ends

## Tasks

### 1. Create webhook receiver endpoint
- File: `app/api/ai-agent/webhook/route.ts`
- **POST** — Receives the structured data from Telnyx AI webhook tool
- This is called BY Telnyx's AI (not by the user) when the AI decides to save caller info
- Payload matches the `body_parameters` from T7.2's webhook tool config:

```typescript
interface AIAgentWebhookPayload {
  caller_name: string;
  callback_number?: string;
  reason: string;
  callback_time?: string;
  age_range?: string;
  state?: string;
  urgency?: string;
  notes?: string;
}
```

- **Security**: This endpoint is called by Telnyx, not by authenticated users. Options:
  - Validate via a shared secret in a custom header (configured in the webhook tool headers)
  - Or validate via Telnyx signature verification if available
  - At minimum: validate the payload shape with Zod, reject malformed requests

### 2. Store the raw webhook data
- On receiving a valid webhook, store the data temporarily so T7.4 can process it into a lead
- Options:
  - Write directly to a `ai_agent_calls` table (new table, simple):
  
```sql
CREATE TABLE ai_agent_calls (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  agent_id UUID NOT NULL REFERENCES auth.users(id),
  telnyx_conversation_id TEXT,
  caller_phone TEXT,
  caller_name TEXT,
  callback_number TEXT,
  reason TEXT,
  callback_time TEXT,
  age_range TEXT,
  state TEXT,
  urgency TEXT DEFAULT 'low',
  notes TEXT,
  transcript TEXT,
  processed BOOLEAN DEFAULT false,
  lead_id UUID REFERENCES leads(id),
  created_at TIMESTAMPTZ DEFAULT NOW()
);

ALTER TABLE ai_agent_calls ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Users can view their AI calls" ON ai_agent_calls
  FOR SELECT USING (agent_id = auth.uid());
```

### 3. Associate webhook calls with agents
- The webhook tool doesn't automatically include which agent the call was for
- Solution: Include the agent_id in the webhook URL as a path parameter or query parameter:
  - Webhook URL format: `https://[domain]/api/ai-agent/webhook?agent_id=[uuid]`
  - When building the assistant config in T7.2, inject the agent's ID into the URL
  - The webhook endpoint extracts agent_id from the query string
  - Validate the agent_id exists and has an active AI assistant

### 4. Telnyx conversation metadata enrichment
- After receiving the webhook data, optionally fetch additional context from Telnyx:
  - Call `getConversations(assistantId)` to find the most recent conversation
  - Call `getTranscript(conversationId)` to get the full conversation transcript
  - Store `telnyx_conversation_id` and `transcript` in `ai_agent_calls`
- This is a best-effort enrichment — if it fails, the webhook data alone is still usable

### 5. Webhook URL configuration
- The webhook URL needs to be the public URL of the Ensurance deployment
- Create a helper that builds the full webhook URL:

```typescript
export function getAIAgentWebhookUrl(agentId: string): string {
  const baseUrl = process.env.NEXT_PUBLIC_APP_URL || process.env.VERCEL_URL;
  return `${baseUrl}/api/ai-agent/webhook?agent_id=${agentId}`;
}
```

- Add `NEXT_PUBLIC_APP_URL` to the environment variables documentation

## Guardrails

- Do NOT require user authentication on the webhook endpoint — Telnyx calls it
- DO validate the payload shape and agent_id
- Do NOT block on transcript fetching — store webhook data first, enrich async
- Return 200 quickly — Telnyx has timeout limits on webhook responses
- Do NOT store sensitive data beyond what's needed for lead creation
