## 9 Pillars

### 1. Objective
Rethink the AI Assistant panel in the quote workspace. Currently it shows live transcription chunks during calls, which is overwhelming and not actionable. Move transcription to background processing and instead surface real-time insights, carrier alerts, and sales coaching tips.

### 2. Scope
- Live transcription moves to a collapsible "raw transcript" section (hidden by default)
- Main panel shows AI-generated insights derived from the transcription
- Insight types: medication detected, condition flagged, carrier recommendation, objection handler
- Insights appear as compact cards, not paragraphs of text
- Existing chat mode (non-call) stays the same

- AI assistant panel (find: `grep -rn "AiAssistant\|ai-assistant\|AssistantPanel" --include="*.tsx" components/quote/`)

**Insight card component:**

```typescript
interface InsightCard {
  id: string;
  type: 'medication' | 'condition' | 'carrier_alert' | 'coaching' | 'info';
  severity: 'critical' | 'warning' | 'info' | 'tip';
  title: string;        // "Metformin detected"
  details: string[];    // ["Type 2 Diabetes likely", "JH: Preferred DB", ...]
  timestamp: Date;
}

function InsightCardComponent({ insight }: { insight: InsightCard }) {
  const iconMap = {
    critical: AlertTriangle,
    warning: AlertCircle, 
    info: Info,
    tip: Lightbulb,
  };
  const colorMap = {
    critical: 'text-destructive border-destructive/30 bg-destructive/5',
    warning: 'text-amber-600 border-amber-300 bg-amber-50',
    info: 'text-blue-600 border-blue-300 bg-blue-50',
    tip: 'text-green-600 border-green-300 bg-green-50',
  };
  const Icon = iconMap[insight.severity];

  return (
    <div className={cn("rounded-lg border p-3 mb-2", colorMap[insight.severity])}>
      <div className="flex items-center gap-2 font-medium text-sm">
        <Icon className="h-4 w-4" />
        {insight.title}
      </div>
      <ul className="mt-1 text-sm space-y-0.5 ml-6">
        {insight.details.map((detail, i) => (
          <li key={i}>{detail}</li>
        ))}
      </ul>
    </div>
  );
}
```

**Processing pipeline:**

Instead of showing raw transcript, process it through the existing coaching/proactive APIs and map results to insight cards:

1. **Transcript comes in** (from Deepgram via call store)
2. **Periodically send to coaching API** (existing, every ~10 seconds of new speech)
3. **Parse coaching response into structured insights:**
   - Detect medication names → look up in carrier data → generate carrier alert cards
   - Detect condition mentions → cross-reference medical matrix → flag carrier eligibility
   - Detect objections (price, timing, uncertainty) → generate coaching tip cards
   - Detect factual info (age, state, smoking status) → generate info cards

4. **Display insights in the panel** (newest first, max 10 visible, older ones collapse)

**Medication/condition detection (client-side enhancement):**

In addition to the AI coaching API, do simple keyword matching on the transcript for immediate feedback:

```typescript
const MEDICATION_KEYWORDS = {
  'metformin': { condition: 'Type 2 Diabetes', carriers: { jh: 'Preferred DB', amam: 'DECLINE' } },
  'insulin': { condition: 'Diabetes (Insulin)', carriers: { jh: 'Preferred DB', amam: 'DECLINE' } },
  'lisinopril': { condition: 'Hypertension', carriers: { foresters: 'ACCEPT', amam: 'ACCEPT' } },
  // ... expand from existing prescription data
};

function detectMedications(transcriptChunk: string): InsightCard[] {
  const insights: InsightCard[] = [];
  const lower = transcriptChunk.toLowerCase();
  
  for (const [med, info] of Object.entries(MEDICATION_KEYWORDS)) {
    if (lower.includes(med)) {
      insights.push({
        type: 'medication',
        severity: 'warning',
        title: `${med.charAt(0).toUpperCase() + med.slice(1)} detected`,
        details: [
          `Likely condition: ${info.condition}`,
          ...Object.entries(info.carriers).map(([carrier, status]) => 
            `${carrier.toUpperCase()}: ${status}`
          ),
        ],
      });
    }
  }
  return insights;
}
```

This gives INSTANT feedback without waiting for the AI coaching API roundtrip.

### 6. Failure Handling
- If coaching API fails, client-side keyword detection still works (degraded but functional)
- If no medications/conditions detected, show generic coaching tips
- If transcript is empty (muted call), show "Waiting for conversation..." message

### 7. Guardrails
- Do NOT remove the raw transcript entirely — agents may want to reference exact words
- Do NOT make the keyword detection the sole insight source — it's a supplement to the AI coaching API
- Do NOT change the existing coaching or proactive insight API contracts
- Do NOT show medical diagnoses — say "likely condition" not "the client has..."
- Keep insights concise — max 4 bullet points per card
