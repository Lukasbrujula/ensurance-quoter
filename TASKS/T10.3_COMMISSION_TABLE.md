# T10.3: Commission Table — Support Unlimited Carriers

**Priority:** Medium — prep for Compulife integration
**Estimate:** 45 min
**Phase:** 10 (Dashboard + UX Polish)

---

## 9 Pillars

### 1. Objective
The commission settings table currently hardcodes 11 carriers. When Compulife integrates, there could be 50-100+ carriers. The table needs to handle arbitrary carrier counts with search, scroll, and bulk editing.

### 2. Scope
- Refactor commission settings table to be data-driven (not hardcoded list)
- Add search/filter for carrier name
- Add virtual scrolling or pagination for large lists
- Add "Set All" bulk action for default rates
- Keep per-carrier override capability

### 3. Dependencies
Find the commission settings component:
```bash
grep -rn "commission\|Commission" --include="*.tsx" components/settings/
cat components/settings/commission-settings*.tsx
```
Check the commission store:
```bash
grep -rn "commission" --include="*.ts" lib/stores/ lib/supabase/
```

### 4. Implementation

**Make carrier list dynamic:**

Currently the table likely maps over a hardcoded array of 11 carriers. Change it to:
1. Fetch carriers from the carriers data source (or in future, from Compulife API response)
2. Merge with saved commission overrides from agent_settings
3. Render dynamically

**Search filter:**
```typescript
const [search, setSearch] = useState('');
const filteredCarriers = carriers.filter(c => 
  c.name.toLowerCase().includes(search.toLowerCase())
);

<Input 
  placeholder="Search carriers..." 
  value={search} 
  onChange={(e) => setSearch(e.target.value)}
  className="max-w-sm mb-4"
/>
```

**Scrollable container:**
For 50+ carriers, wrap the table in a ScrollArea with a fixed height:
```typescript
<ScrollArea className="h-[500px]">
  <Table>
    {filteredCarriers.map(carrier => (
      <TableRow key={carrier.id}>
        {/* carrier name, FY%, RN%, actions */}
      </TableRow>
    ))}
  </Table>
</ScrollArea>
```

**Bulk actions:**
```typescript
<div className="flex gap-2 mb-4">
  <Button variant="outline" size="sm" onClick={() => setAllFY(defaultFY)}>
    Set All FY% to {defaultFY}%
  </Button>
  <Button variant="outline" size="sm" onClick={() => setAllRN(defaultRN)}>
    Set All RN% to {defaultRN}%
  </Button>
  <Button variant="outline" size="sm" onClick={resetAllToDefaults}>
    Reset All to Defaults
  </Button>
</div>
```

**"Only show overrides" toggle:**
When there are 100+ carriers, agents may want to see only the ones they've customized:
```typescript
<div className="flex items-center gap-2">
  <Switch checked={showOverridesOnly} onCheckedChange={setShowOverridesOnly} />
  <Label>Show only carriers with custom rates</Label>
</div>
```

### 5. Success Criteria
- [ ] Commission table renders correctly with 11 carriers (current)
- [ ] Commission table handles 50+ carriers without performance issues
- [ ] Search filter narrows the carrier list by name
- [ ] ScrollArea prevents the page from becoming infinitely long
- [ ] "Set All" buttons apply default rates to all carriers
- [ ] "Show overrides only" toggle works
- [ ] Per-carrier override still works (edit individual FY%/RN%)
- [ ] Saved overrides persist across page reloads
- [ ] `npx tsc --noEmit` passes

### 6. Failure Handling
- If carrier list is empty (no data loaded), show "No carriers available" with note about Compulife
- If save fails for bulk update, show error and don't clear dirty state
- If search returns no results, show "No carriers match" message

### 7. Guardrails
- Do NOT change how commission data is stored in Supabase (same JSONB format)
- Do NOT break the existing 11-carrier experience
- Do NOT add virtual scrolling library — ScrollArea is sufficient for 100 carriers
- Keep the debounced save pattern from the existing implementation

### 8. Human Checkpoint
- Open commission settings, verify all 11 carriers show
- Use search to filter to "Foresters" — should show only that row
- Click "Set All FY%" — verify all rows update
- Save, reload, verify persistence
