# T8.5: Usage Dashboard — Call Counts, Minutes, Cost Estimates

**Priority:** Medium — operational visibility for agents
**Estimate:** 1.5 hrs
**Phase:** 8 (Agent Management + Transcripts + Usage)
**Depends on:** T8.2 (usage API endpoint exists)

---

## Context

Agents need to know how much their AI agents are being used — how many calls, how many minutes, and roughly what it's costing them. This is similar to the usage page in Claude or OpenAI's dashboard. It helps agents understand ROI and manage costs.

## 9 Pillars

### 1. Objective
Build a usage dashboard showing per-agent and total usage statistics with time-range filtering and cost estimates.

### 2. Scope
- Usage section: either a tab on `/agents` page or a dedicated `/settings/usage` page
- Per-agent breakdown: calls, minutes, estimated cost
- Totals: aggregate across all agents
- Time range filter: This month, Last 30 days, Last 90 days, All time
- Simple cost estimation based on Telnyx per-minute rates
- Visual: bar chart or table (pick based on existing charting in codebase)

### 3. Dependencies
- T8.2 completed (GET /api/agents/usage endpoint)
- Existing charting: check if Recharts is used anywhere (`grep -rn "recharts\|LineChart\|BarChart" --include="*.tsx"`)
- Existing settings page layout (if placing usage in settings)
- shadcn/ui: Card, Table, Badge, Select, Skeleton

### 4. Implementation

**Placement decision:**
- **Option A: Tab on /agents page** — "My Agents" tab (list) + "Usage" tab (dashboard). Keeps everything in one place.
- **Option B: /settings/usage page** — Separate settings section. Better if usage will expand to cover non-agent costs (Deepgram, OpenAI, PDL) later.
- **Recommendation:** Option A for now (tab on /agents). Can move to settings later when more services have usage tracking.

**Summary cards:**
Three stat cards at the top:
- Total Calls (number, with delta from previous period if available)
- Total Minutes (formatted to 1 decimal)
- Estimated Cost (calculated from minutes x rate)

**Cost estimation:**
- Telnyx AI agent calls: approximately $0.05/minute (verify against current Telnyx pricing)
- Store the per-minute rate as a constant, not hardcoded in multiple places
- Display as "Estimated" with a tooltip: "Based on Telnyx per-minute rate of $X.XX. Actual charges may vary."
- This is an approximation — not a billing system

```typescript
const TELNYX_AI_COST_PER_MINUTE = 0.05; // USD, verify against Telnyx pricing

function estimateCost(minutes: number): number {
  return Math.round(minutes * TELNYX_AI_COST_PER_MINUTE * 100) / 100;
}
```

**Per-agent table:**
- Columns: Agent Name, Status (badge), Calls, Minutes, Estimated Cost
- Sortable by any column
- Link agent name to /agents/[id]
- If an agent has 0 calls, still show it (with dashes or zeros)

**Time range filter:**
- Dropdown: "This Month", "Last 30 Days", "Last 90 Days", "All Time"
- Default: "This Month"
- Filter is passed to the usage API endpoint as query params
- API should support `since` and `until` date params (or add them in this task if T8.2 didn't include them)

**Monthly trend chart (stretch goal):**
- If Recharts is already in the project, add a simple bar chart showing calls per day (last 30 days) or per week (last 90 days)
- If no charting library is present, skip this and just show the table. Don't install a new library.
- Chart should be clean and minimal — no over-styling

**Empty state:**
If no calls have been made yet:
- "No usage data yet"
- "Once your AI agents start handling calls, usage statistics will appear here"

### 5. Success Criteria
- [ ] Usage dashboard renders with summary cards (calls, minutes, estimated cost)
- [ ] Per-agent breakdown table shows each agent's stats
- [ ] Time range filter changes the displayed data
- [ ] Cost estimation uses a configurable per-minute rate
- [ ] Table is sortable
- [ ] Agent names link to detail page
- [ ] Empty state renders when no usage data exists
- [ ] Data refreshes when time range changes (no stale data)
- [ ] Responsive: cards stack on mobile, table scrolls horizontally
- [ ] `npx tsc --noEmit` passes

### 6. Failure Handling
- If usage API fails, show error state with "Retry" button
- If one agent's data is missing, show the row with "N/A" values (don't break the table)
- If cost estimation rate needs updating, it's in one constant (easy to change)

### 7. Guardrails
- Do NOT build a real billing system — this is estimation only
- Do NOT call Telnyx's billing API — use our own call log data for minutes
- Do NOT install new charting libraries if Recharts isn't already in the project
- Do NOT show usage data from other users (RLS + API-level checks)
- Keep the design simple — this is an internal operations tool, not a customer-facing dashboard

### 8. Human Checkpoint
- Visual: do the summary cards and table look clean and consistent?
- Data: after making some AI agent test calls, does usage data appear?
- Filter: does changing the time range update the numbers?
- Mobile: does the layout work on small screens?
