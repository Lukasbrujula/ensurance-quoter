# T7.5: Settings > Integrations — AI Agent Config Panel

**Priority:** Medium — agent-facing control panel  
**Estimate:** 1 hr  
**Phase:** 7 (Telnyx AI Agent)  
**Depends on:** T7.2 (assistant config exists to create/update)

---

## Context

The /settings/integrations page is currently a Phase 5 placeholder (Coming Soon). This task replaces it with a functional AI Agent configuration panel. Since Ensurance currently supports one integration (Telnyx AI Agent), the page focuses on that. The layout should support adding more integrations later (Compulife, SendGrid, etc.) but only the AI Agent section is built now.

## Tasks

### 1. Replace the placeholder with an integrations layout

The page should show a list of integration cards, with AI Agent as the first (and currently only) active one:

```
/settings/integrations
  AI Voice Agent (Telnyx)      [Active / Inactive toggle]
  Compulife Pricing API         [Coming Soon badge]
  Email Service (SendGrid)      [Coming Soon badge]
  Lead Enrichment (PDL)         [Coming Soon badge]
```

Each integration is a Card with: icon, name, brief description, status badge, and Configure button.

### 2. AI Agent configuration panel

When the agent clicks Configure on the AI Voice Agent card (or when expanded inline), show:

**Enable/Disable Toggle**
- Large, prominent switch at the top
- When enabling for the first time: creates the Telnyx assistant via ensureAIAgent() and stores the ID
- When disabling: does not delete the assistant, just sets ai_agent_enabled = false
- When re-enabling: updates the existing assistant (does not create a new one)

**Status Indicator**
- Green dot + Active when enabled and assistant exists
- Gray dot + Inactive when disabled
- Red dot + Error if assistant creation failed

**Phone Number**
- Show the Telnyx phone number currently assigned (from TELNYX_CALLER_NUMBER env or stored in agent_settings)
- Note: Calls to this number that go unanswered will be handled by the AI agent
- If no number is configured, show a warning

**AI Personality Preview**
- Read-only card showing:
  - Agent name (from user profile)
  - Business name (from user profile or a new field)
  - Greeting message preview (what the AI says when it answers)
  - A brief description of what the AI collects
- Customize button for editing (stretch goal, can defer)

**Test Call Button**
- Make a Test Call button that opens the Telnyx AI agent widget inline (WebRTC browser call)
- This lets the agent hear exactly what callers will experience
- Uses telephony_settings.supports_unauthenticated_web_calls capability
- Implementation: embed a call to the assistant using the Telnyx AI Agent Widget URL https://ai.telnyx.com/widget/{assistant_id} or use existing WebRTC infrastructure

**Recent AI Calls**
- Show last 5 AI-handled calls as a compact list:
  - Caller name, date/time, reason, status (lead created / error)
  - Link to the lead in CRM
- Pull from leads table where source = ai_agent plus call_logs

### 3. API endpoints

Create or extend settings endpoints:

**Extend PUT /api/settings:**
- Add ai_agent_enabled boolean and ai_agent_id string to the settings upsert
- When ai_agent_enabled changes from false to true: call ensureAIAgent() and store the returned assistant ID
- When changing from true to false: update settings only (do not delete the Telnyx assistant)

Or create dedicated endpoints:
- POST /api/telnyx/ai-agent/enable creates/updates assistant, stores ID, sets enabled
- POST /api/telnyx/ai-agent/disable sets enabled to false

### 4. Loading and error states

- Show skeleton while loading agent_settings
- Show toast on enable/disable success
- Show error toast if Telnyx API call fails, with Retry action
- If the stored assistant_id is invalid (Telnyx returns 404), offer to Recreate Agent

## Guardrails

- Do NOT delete the Telnyx assistant when disabling. Agent might re-enable.
- Do NOT expose the Telnyx API key or assistant ID to the client. Keep server-side.
- Do NOT build a full agent builder with prompt editing. That is a future feature.
- Do NOT modify the settings sidebar navigation. Just replace the placeholder content.
- Follow existing settings page patterns (commission settings as reference for layout, loading states, save behavior).

## Failure Handling

- If Telnyx API is down during enable, show error and do not change the toggle state
- If assistant_id is stored but Telnyx returns 404, offer Recreate (delete ID from settings, create new)
- If phone number is not configured, show warning but still allow enabling (agent can configure number later)
