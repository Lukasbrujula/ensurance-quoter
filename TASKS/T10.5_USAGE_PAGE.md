# T10.5: Usage Page — Claude-Style Platform Metrics

**Priority:** Medium — operational visibility into platform costs
**Estimate:** 1.5 hrs
**Phase:** 10 (Dashboard + UX Polish)

---

## 9 Pillars

### 1. Objective
Build `/settings/usage` page modeled after Claude's usage page (see reference screenshot). Shows the agent's platform consumption: phone numbers, call minutes, quotes generated, enrichment lookups, and estimated costs. Separate from the per-agent usage on the `/agents` page — this is platform-wide.

### 2. Scope
- New route: `app/settings/usage/page.tsx` (or transform existing placeholder)
- Add "Usage" to settings sidebar navigation
- Sections: My Numbers, Calling Minutes, Quotes, Enrichment, Cost Summary
- Time range filter (This Month, Last 30 Days, All Time)
- Visual progress bars or usage meters (Claude-style)

### 3. Dependencies
- Settings layout/sidebar (find: `grep -rn "SettingsLayout\|settings.*nav\|sidebar" --include="*.tsx" app/settings/ components/settings/`)
- Existing agent usage from Phase 8 T8.5 (find: `grep -rn "usage\|Usage" --include="*.tsx" app/agents/`)
- Call logs table for minutes data
- Quote response data for quote count
- Enrichment data for lookup count
- AI agent calls for AI minutes

**My Numbers section:**
- List phone numbers from `TELNYX_CALLER_NUMBER` env var and any stored in agent_settings or ai_agents
- Show status (Active/Inactive)
- Show capability (Inbound only, Outbound only, Both)
- "Add Number" button (can link to Telnyx dashboard or be a future feature)

**Calling section with progress bars:**
Use visual bars similar to Claude's usage display:
```typescript
function UsageBar({ used, label }: { used: number; label: string }) {
  return (
    <div className="space-y-1">
      <div className="flex justify-between text-sm">
        <span className="text-muted-foreground">{label}</span>
        <span className="font-medium">{used} min</span>
      </div>
      <div className="h-2 bg-muted rounded-full overflow-hidden">
        <div 
          className="h-full bg-primary rounded-full" 
          style={{ width: `${Math.min((used / 100) * 100, 100)}%` }}
        />
      </div>
    </div>
  );
}
```

Split calling into:
- **Outbound minutes**: Agent-initiated calls via WebRTC (from call_logs where direction = 'outbound')
- **Inbound minutes**: Calls answered by agent (from call_logs where direction = 'inbound' and NOT ai-handled)
- **AI Agent minutes**: Calls handled by AI agent (from ai_agent_calls)

**Platform usage section:**
Simple count cards:
- Quotes generated: COUNT from quotes table this period
- Enrichment lookups: COUNT from enrichments table this period
- AI chat messages: Approximation from chat API usage (or count from a new counter)
- AI coaching prompts: Count from coaching-related call activities

**Estimated costs table:**
Use configurable rates (same pattern as T8.5):

### 5. Success Criteria
- [ ] `/settings/usage` page renders with all sections
- [ ] "Usage" appears in settings sidebar navigation
- [ ] My Numbers shows configured phone numbers with status
- [ ] Calling section breaks down outbound, inbound, and AI minutes
- [ ] Platform usage shows quote count, enrichment count, AI usage
- [ ] Cost table estimates per-service costs with total
- [ ] Time range filter changes displayed data
- [ ] Visual bars/meters match the Claude usage page aesthetic
- [ ] Responsive layout (sections stack on mobile)
- [ ] `npx tsc --noEmit` passes

### 6. Failure Handling
- If usage API fails, show error with retry
- If no data for a metric, show 0 (not an error)
- If cost rates need updating, they're in one constant object
- If phone number data isn't available, show "No numbers configured" with setup link

### 7. Guardrails
- Do NOT build real billing integration — this is estimation only
- Do NOT call external APIs for usage data (Telnyx billing API etc.) — use our own tables
- Do NOT make this a real-time dashboard (load on page visit, no polling)
- Follow the existing settings page layout pattern exactly
- Cost rates are approximations — label them clearly as estimates

### 8. Human Checkpoint
- Does the page look and feel like Claude's usage page?
- Do the numbers match reality (check against database)?
- Does the time range filter work?
- Is the cost estimation reasonable?
