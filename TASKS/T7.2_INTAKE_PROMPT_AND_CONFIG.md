## Context

This task creates the system prompt and default configuration for the Ensurance AI intake agent. The agent's job is simple: answer when the insurance agent can't, collect basic information from the caller, and end the call politely. It must NEVER give insurance advice, quotes, or product recommendations — that's a regulatory minefield.

1. `VOICE_AI_PROMPT_ENGINEERING_REFERENCE.md` — Production voice prompt patterns. Key principles:

## Tasks

### 1. Write the insurance intake system prompt
- File: `lib/telnyx/ai-prompts.ts`
- Export a function that generates the system prompt with agent-specific details:

```typescript
export function buildInsuranceIntakePrompt(agentName: string, agencyName?: string): string
```

**Prompt requirements:**

The AI must:
- Greet the caller professionally: "Hi, you've reached [Agent Name]'s office. They're not available right now, but I can help get you started."
- Collect these fields (goal-based, not script-based):
  - Caller's name
  - Best callback number (if different from caller ID)
  - Why they're calling (general inquiry, need a quote, have a policy question, etc.)
  - Best time for a callback
  - Optionally: age range, state of residence (if they volunteer it naturally)
- End the call gracefully: "Great, I'll make sure [Agent Name] gets this information. They'll give you a call back [at the time you mentioned]. Have a great day!"

The AI must NEVER:
- Give insurance advice or recommendations
- Quote prices or estimate premiums
- Recommend specific carriers or products
- Discuss medical conditions in detail
- Promise coverage or eligibility
- Say "I'm an AI" unprompted (but be honest if directly asked)
- Use insurance jargon the caller didn't introduce

**Caller paths to handle explicitly:**
1. **New inquiry** — wants a quote or info → collect basics, promise callback
2. **Existing client** — has a policy question → collect name + policy concern, promise callback
3. **Urgent matter** — needs immediate help → "I understand this is urgent. Let me get your details so [Agent] can prioritize your call."
4. **Wrong number / spam** — end call politely
5. **Wants to speak to someone NOW** — "I understand. [Agent] isn't available right now, but I can make sure they call you back as soon as possible. Can I get your name?"

### 2. Define the default assistant configuration
- File: `lib/telnyx/ai-config.ts`
- Export a function that builds the full `TelnyxAssistantCreateDto`:

```typescript
export function buildInsuranceAssistantConfig(
  agentName: string,
  agencyName?: string,
  webhookUrl?: string,
): TelnyxAssistantCreateDto
```

**Configuration (from TELNYX_WORKING_CONFIG.md — follow exactly):**

```typescript
{
  name: `Ensurance AI - ${agentName}`,
  model: 'Qwen/Qwen3-235B-A22B',  // ONLY model that works correctly
  instructions: buildInsuranceIntakePrompt(agentName, agencyName),
  greeting: `Hi, you've reached ${agentName}'s office. They're not available right now, but I can take some information so they can call you back. How can I help?`,
  transcription: {
    model: 'deepgram/nova-2',
    language: 'en',
  },
  voice_settings: {
    voice: 'Telnyx.NaturalHD.astra',
  },
  telephony_settings: {
    supports_unauthenticated_web_calls: true,
  },
  widget_settings: {
    theme: 'light',
  },
  enabled_features: ['telephony'],
  tools: webhookUrl ? [{
    type: 'webhook' as const,
    name: 'save_caller_info',
    description: 'Save the collected caller information when the conversation is ending',
    webhook: {
      url: webhookUrl,
      method: 'POST' as const,
      body_parameters: {
        type: 'object',
        properties: {
          caller_name: { type: 'string', description: 'The caller full name' },
          callback_number: { type: 'string', description: 'Best phone number to call back' },
          reason: { type: 'string', description: 'Why they are calling (brief summary)' },
          callback_time: { type: 'string', description: 'When the caller prefers to be called back' },
          age_range: { type: 'string', description: 'Caller approximate age if mentioned' },
          state: { type: 'string', description: 'Caller state of residence if mentioned' },
          urgency: { type: 'string', description: 'low, medium, or high based on caller tone and request' },
          notes: { type: 'string', description: 'Any other relevant details from the conversation' },
        },
        required: ['caller_name', 'reason'],
      },
    },
  }] : [],
}
```

### 3. Create/update assistant API route
- File: `app/api/ai-agent/route.ts`
- **POST** — Create or update the agent's AI assistant
  - If agent has no `telnyx_ai_assistant_id`: create new assistant via `createAssistant()`
  - If agent already has one: update existing via `updateAssistant()`
  - Store the assistant ID in `agent_settings.telnyx_ai_assistant_id`
  - Return the assistant config
- **GET** — Get current AI agent status (enabled/disabled, assistant ID, last call info)
- **DELETE** — Disable and delete the AI assistant
  - Call `deleteAssistant()` on Telnyx
  - Clear `telnyx_ai_assistant_id` and set `telnyx_ai_enabled = false`
- Auth-guarded, rate-limited (consistent with existing API patterns)

### 4. Enable/disable toggle API
- File: `app/api/ai-agent/toggle/route.ts`
- **PUT** — Toggle AI agent on/off
  - Validates assistant exists before enabling
  - Updates `telnyx_ai_enabled` in agent_settings
  - When enabling: verify the assistant still exists on Telnyx (GET), recreate if needed
  - When disabling: don't delete the assistant — just flag as disabled

## Guardrails

- The AI agent must NEVER give insurance advice — regulatory requirement
- Do NOT use the Llama model — only `Qwen/Qwen3-235B-A22B`
- Do NOT add a hangup tool — it breaks WebRTC (reference gotcha #2)
- Do NOT override Telnyx defaults for voice_speed, noise_suppression, time_limit_secs
- Do NOT make the webhook URL hardcoded — it needs to be configurable per deployment
- Keep the prompt under 2000 tokens — shorter is better for voice AI latency
