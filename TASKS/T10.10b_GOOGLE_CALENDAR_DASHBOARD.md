#### File 1: `app/api/dashboard/calendar/route.ts`

```typescript
// GET /api/dashboard/calendar?days=7
// 
// 1. requireUser()
// 2. Get Ensurance follow-ups from leads table (next N days)
// 3. Check if Google Calendar is connected
// 4. If connected, get Google Calendar events (next N days)
// 5. Merge and deduplicate:
//    - Google events with ensurance_lead_id match to a follow-up → show as one event (enriched)
//    - Google events without ensurance metadata → show as "external" events (gray)
//    - Ensurance follow-ups without google_event_id → show as "unsynced" (still visible)
// 6. Sort by start time
// 7. Return unified event list

interface CalendarEvent {
  id: string;
  type: 'follow_up' | 'ai_callback' | 'external';  // source
  title: string;                  // "Callback: John Smith" or "Team standup"
  startTime: string;              // ISO
  endTime?: string;               // ISO
  leadId?: string;                // if linked to a lead
  leadName?: string;              // for display
  source: 'ensurance' | 'google' | 'both';  // where it came from
  googleEventId?: string;         // if synced to Google
  note?: string;                  // follow-up note or event description
}
```

The merge logic:
```typescript
function mergeEvents(
  followUps: LeadFollowUp[],
  googleEvents: GoogleCalendarEvent[]
): CalendarEvent[] {
  const merged: CalendarEvent[] = [];
  const matchedGoogleIds = new Set<string>();

  // First: map Ensurance follow-ups
  for (const fu of followUps) {
    const event: CalendarEvent = {
      id: fu.leadId,
      type: fu.source === 'ai_agent' ? 'ai_callback' : 'follow_up',
      title: `${fu.source === 'ai_agent' ? 'AI Callback' : 'Follow-up'}: ${fu.leadName}`,
      startTime: fu.followUpDate,
      leadId: fu.leadId,
      leadName: fu.leadName,
      source: fu.googleEventId ? 'both' : 'ensurance',
      googleEventId: fu.googleEventId,
      note: fu.followUpNote,
    };
    merged.push(event);
    
    if (fu.googleEventId) matchedGoogleIds.add(fu.googleEventId);
  }

  // Second: add Google events that aren't already matched
  for (const ge of googleEvents) {
    if (matchedGoogleIds.has(ge.id!)) continue;  // Already shown via follow-up
    
    // Check if it's an Ensurance-created event we missed
    const ensuranceLeadId = ge.extendedProperties?.private?.ensurance_lead_id;
    if (ensuranceLeadId) continue;  // Should have been matched above

    // External event — show in gray
    merged.push({
      id: `google-${ge.id}`,
      type: 'external',
      title: ge.summary || 'Untitled event',
      startTime: ge.start?.dateTime || ge.start?.date || '',
      endTime: ge.end?.dateTime || ge.end?.date || '',
      source: 'google',
      googleEventId: ge.id,
      note: ge.description,
    });
  }

  return merged.sort((a, b) => 
    new Date(a.startTime).getTime() - new Date(b.startTime).getTime()
  );
}
```

#### Sync Hooks — Modify Existing Files

**Hook 1: AI lead processor** (`lib/telnyx/ai-lead-processor.ts`)

After the AI agent creates a lead with a follow_up_date:
```typescript
// EXISTING: lead is created with follow_up_date
const lead = await createLead({ ...data, follow_up_date: callbackDate });

// NEW: sync to Google Calendar (non-blocking)
import { createCalendarEvent } from '@/lib/google/calendar-service';

createCalendarEvent(agentId, {
  title: `AI Callback: ${lead.name || 'Unknown caller'}`,
  description: `Booked by AI voice agent. Reason: ${data.reason || 'Not specified'}`,
  startTime: callbackDate,
  leadId: lead.id,
}).then(googleEventId => {
  if (googleEventId) {
    // Update lead with Google event ID
    updateLead(lead.id, agentId, { google_event_id: googleEventId });
  }
}).catch(err => {
  console.error('[Google Calendar] Failed to create AI callback event:', err);
});
```

if (newFollowUpDate && newFollowUpDate !== oldFollowUpDate) {
  if (lead.google_event_id) {
    // Update existing Google event
    updateCalendarEvent(agentId, lead.google_event_id, {
      startTime: newFollowUpDate,

    createCalendarEvent(agentId, {

if (!newFollowUpDate && lead.google_event_id) {
  deleteCalendarEvent(agentId, lead.google_event_id).catch(console.error);
  updateLead(lead.id, agentId, { google_event_id: null });
}

**Important: All calendar operations are fire-and-forget.**

Every Google Calendar call is wrapped in `.catch()` and never blocks the primary operation. If Google is down, the follow-up still saves, the AI lead is still created, the dashboard still loads (just without Google events).

### 6. Failure Handling
- If `/api/dashboard/calendar` fails, show error with retry button in calendar view
- If Google token is expired and refresh fails, show events as "ensurance only" + "Reconnect" link
- If a lead linked to a Google event is deleted, the Google event stays (agent can delete manually)
- If Google Calendar returns rate limit error, back off and show Ensurance events only
- Empty weeks show "No events" per day, not an error

### 7. Guardrails
- Do NOT build a full month grid calendar (week view is more useful for agents)
- Do NOT make Google Calendar required (everything works without it)
- Do NOT modify or delete external Google events (only touch events Ensurance created)
- Do NOT block lead operations if Google sync fails
- Do NOT poll Google Calendar from the client (fetch on page load and week navigation only)
- Keep the calendar view lightweight — max 50 events per week
- External Google events are read-only in Ensurance (no editing)
