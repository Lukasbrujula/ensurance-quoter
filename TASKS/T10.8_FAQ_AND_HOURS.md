# T10.8: Agent FAQ Knowledge Base + Business Hours

**Priority:** Medium — makes AI agents significantly more useful
**Estimate:** 1 hr
**Phase:** 10 (Dashboard + UX Polish)
**Depends on:** T10.7 (templates exist)

---

## 9 Pillars

### 1. Objective
Expand the AI agent configuration with two features: a FAQ knowledge base (the agent can answer common questions) and business hours (the agent behaves differently during vs outside business hours).

### 2. Scope
- FAQ editor: add Q&A pairs that the AI agent can reference during calls
- Business hours: set weekly schedule, AI agent greeting changes based on open/closed
- Both stored on the ai_agents table (new columns or JSONB)
- Update the Telnyx assistant system prompt to include FAQ and hours context

### 3. Dependencies
- AI agent detail page from Phase 8 T8.4 (find: `grep -rn "agent.*detail\|AgentDetail" --include="*.tsx" app/agents/`)
- AI prompt builder from T7.2 / T10.7 (find: `cat lib/telnyx/ai-prompts.ts`)
- ai_agents table schema

interface FAQEntry {

```typescript
interface BusinessHours {
  timezone: string;  // "America/New_York"

    tuesday:   { open: string; close: string } | null;

    thursday:  { open: string; close: string } | null;
    friday:    { open: string; close: string } | null;
    saturday:  { open: string; close: string } | null;
    sunday:    { open: string; close: string } | null;
  };
}
```

**Inject FAQ and hours into the system prompt:**

When building or updating the assistant's system prompt, append FAQ context:

```typescript
function appendFAQContext(basePrompt: string, faqEntries: FAQEntry[]): string {
  if (faqEntries.length === 0) return basePrompt;
  
  let faqSection = '\n\nYou can answer these common questions if asked:\n';
  faqEntries.forEach(entry => {
    faqSection += `\nQuestion: ${entry.question}\nAnswer: ${entry.answer}\n`;
  });
  faqSection += '\nIf asked something not covered above, let the caller know that the agent can help with that when they call back.';
  
  return basePrompt + faqSection;
}

function appendHoursContext(basePrompt: string, hours: BusinessHours | null): string {
  if (!hours) return basePrompt;
  
  const hoursText = formatBusinessHoursForPrompt(hours);
  return basePrompt + `\n\nBusiness hours: ${hoursText}. If a caller asks about hours, share this information.`;
}
```

**Auto-switch greeting based on hours:**
When creating/updating the Telnyx assistant, check if current time is within business hours. If not, use the `after_hours_greeting` instead of the regular greeting. This can be done by updating the assistant periodically or by including time-awareness in the prompt.

Simpler approach: include BOTH greetings in the system prompt and let the AI decide:
```
If the current time is outside business hours (Mon-Fri 9AM-5PM ET), use this greeting: [after hours greeting]
Otherwise, use this greeting: [regular greeting]
```

- [ ] `npx tsc --noEmit` passes

### 6. Failure Handling
- If Telnyx update fails after FAQ change, show error toast but save to database (retry sync later)
- If FAQ makes the prompt too long (>4000 words), warn the agent to reduce entries
- If business hours timezone is invalid, default to UTC

### 7. Guardrails
- Do NOT make the FAQ prompt injection exceed 1500 words (voice AI constraint)
- Do NOT attempt real-time hour-based greeting switching (too complex for MVP — let the AI decide from prompt context)
- Do NOT require FAQ or hours to be set (both are optional enhancements)
- Keep the FAQ editor simple — no rich text, no categories, just Q&A pairs

### 8. Human Checkpoint
- Add 3 FAQ entries, save, make a test call, ask one of the questions
- Set business hours, call outside hours, verify the greeting changes
- Verify FAQ answers are accurate and natural-sounding
