# T10.1: Date Picker — Year Jump + Manual Input for DOB

**Priority:** High — current UX is broken for older clients
**Estimate:** 30 min
**Phase:** 10 (Dashboard + UX Polish)

---

## 9 Pillars

### 1. Objective
Fix the date of birth picker so agents can quickly input dates for elderly clients. Currently requires clicking through months one at a time — a 70-year-old's DOB would take 840 clicks. Need year jumping and direct text input.

### 2. Scope
- Wherever DOB is selected in the app (lead creation, lead detail, intake form)
- Add year dropdown/jump (click the year header to get a year grid)
- Add month dropdown/jump
- Allow manual text input (type "03/15/1955" directly)
- Keep the calendar picker as an option (not remove it)

### 3. Dependencies

```
Check which shadcn/ui calendar component is being used:

cat components/ui/calendar.tsx | head -20
```
The shadcn calendar uses `react-day-picker` under the hood.

### 4. Implementation

**Approach: Enhance the existing calendar with navigation + add text input**

The shadcn/ui Calendar component wraps react-day-picker. react-day-picker supports `captionLayout="dropdown"` which adds month/year dropdowns natively:

```typescript
<Calendar
  mode="single"
  selected={date}
  onSelect={setDate}
  captionLayout="dropdown"        // Adds month + year dropdowns
  fromYear={1920}                  // Earliest selectable year
  toYear={new Date().getFullYear()} // Latest selectable year
  defaultMonth={date || new Date(1970, 0)} // Start view at a reasonable decade
/>
```

**Add text input alongside the calendar:**

Wrap the date picker in a Popover (if not already) with a text input that accepts typed dates:

```typescript
<div className="flex gap-2">
  <Input
    type="text"
    placeholder="MM/DD/YYYY"
    value={inputValue}
    onChange={(e) => {
      setInputValue(e.target.value);
      // Parse on valid date format
      const parsed = parseDate(e.target.value); // handles MM/DD/YYYY, YYYY-MM-DD, etc.
      if (parsed && isValid(parsed)) {
        setDate(parsed);
      }
    }}
    onBlur={() => {
      // Format to consistent display on blur
      if (date) setInputValue(format(date, 'MM/dd/yyyy'));
    }}
  />
  <Popover>
    <PopoverTrigger asChild>
      <Button variant="outline" size="icon">
        <CalendarIcon className="h-4 w-4" />
      </Button>
    </PopoverTrigger>
    <PopoverContent>
      <Calendar
        mode="single"
        selected={date}
        onSelect={(d) => {
          setDate(d);
          setInputValue(d ? format(d, 'MM/dd/yyyy') : '');
        }}
        captionLayout="dropdown"
        fromYear={1920}
        toYear={new Date().getFullYear()}
      />
    </PopoverContent>
  </Popover>
</div>
```

### 5. Success Criteria
- [ ] Calendar shows month AND year dropdowns (not just month arrows)
- [ ] Year dropdown allows jumping to any year from 1920 to current
- [ ] Agent can type a date directly into a text input field
- [ ] Typed dates parse correctly for MM/DD/YYYY and YYYY-MM-DD formats
- [ ] Calendar and text input stay in sync (change one updates the other)
- [ ] All DOB fields in the app use the improved picker (lead creation, lead detail, intake form)
- [ ] `npx tsc --noEmit` passes

### 6. Failure Handling
- If `captionLayout="dropdown"` isn't supported by the installed react-day-picker version, implement a custom year selector above the calendar
- If text input produces an invalid date, show inline error and don't update the calendar
- If the calendar component is customized heavily, adapt the approach to fit

### 7. Guardrails
- Do NOT modify `components/ui/calendar.tsx` directly if possible — extend/wrap it
- Do NOT remove the calendar picker (some agents prefer clicking)
- Do NOT change date storage format (keep ISO 8601 in the database)
- Keep the text input simple — no complex date mask library needed

### 8. Human Checkpoint
- Open lead creation, try to set DOB for someone born in 1955
- Should be achievable in under 5 seconds (type or year-jump)
- Try typing "03/15/1955" directly — should work
