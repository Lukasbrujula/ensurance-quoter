# T7.6: CRM Integration — AI-Collected Lead Badges + Call Log Tagging

**Priority:** Medium — visual polish that makes AI leads distinguishable  
**Estimate:** 45 min  
**Phase:** 7 (Telnyx AI Agent)  
**Depends on:** T7.4 (leads are being created with source 'ai_agent')

---

## Context

Once the AI agent is creating leads in the CRM (T7.4), agents need to quickly identify which leads came from the AI vs. manual entry, CSV import, etc. This task adds visual indicators throughout the leads experience. The data is already there (`source: 'ai_agent'` on the lead, call logs with AI transcripts) — this task just surfaces it.

## Tasks

### 1. Source badge on leads list

In the leads list table, add a visual indicator for AI-collected leads:

- **Badge design:** Small pill/badge next to the lead's name or in a "Source" column
  - `ai_agent` → 🤖 "AI" badge (purple or blue accent)
  - `manual` → No badge (default)
  - `csv_import` → "CSV" badge (gray)
  - Future sources can be added as more integrations come online
- **Position:** Either as a new "Source" column or as a subtle badge in the Name column
- **Filter:** Add `source` as a filter option in the leads list — "Show: All / AI Agent / Manual / CSV Import"

### 2. AI call indicator on lead detail

On the lead detail page, if the lead was created by the AI agent:

- Show a subtle info banner or badge at the top: "🤖 This lead was collected by your AI voice agent on [date]"
- Include the callback preference: "Caller requested callback: [preference]"
- Include urgency flag if applicable: "⚠️ Caller indicated this is urgent"
- This info comes from the lead's `notes` field (populated in T7.4)

### 3. Call log tagging

In the call log viewer on the lead detail page, distinguish AI-handled calls from agent calls:

- **AI calls:** Show a "🤖 AI Agent" tag/badge next to the call entry
- **Agent calls:** Show as normal (or with a "👤 Agent" tag for consistency)
- **Detection:** AI calls can be identified by:
  - The call log's transcript containing AI agent conversation patterns
  - A new field on call_logs: consider adding `handled_by` ('agent' | 'ai_agent') — or use the activity_logs `details.handled_by` from T7.4
  - The lead's source being 'ai_agent' combined with the call being the first inbound call

### 4. Activity timeline integration

AI agent events should already appear in the activity timeline (from T7.4 logging). Verify:
- "AI agent handled inbound call" entry shows with a robot/AI icon
- Entry is expandable to show caller name, reason, callback preference
- Transcript link if available

### 5. Leads list sort integration

AI-collected leads with follow-ups should surface naturally via the existing follow-up sort (Phase 6 T6.7). Verify:
- AI leads with auto-scheduled follow-ups appear when sorted by follow-up date
- Overdue AI follow-ups show the red indicator

---

## Success Criteria

- [ ] AI-collected leads show a "🤖 AI" or "AI Agent" badge in the leads list
- [ ] Source filter works (show only AI-collected leads, only manual, only CSV, or all)
- [ ] Lead detail shows AI collection banner with callback preference and urgency
- [ ] Call logs for AI calls are tagged as AI-handled
- [ ] Activity timeline shows AI agent call entries correctly
- [ ] AI leads with follow-ups surface in follow-up sort/filter
- [ ] Non-AI leads are completely unaffected
- [ ] `npx tsc --noEmit` passes

## Dependencies

- T7.4 completed (leads exist with `source: 'ai_agent'`, call logs and activity logs created)
- Leads list component
- Lead detail component  
- Call log viewer component
- Activity timeline component (Phase 6 T6.8)
- Existing filter/sort infrastructure on leads list (Phase 6 T6.6)

## Guardrails

- Do NOT modify how existing (non-AI) leads display — only add indicators for AI leads
- Do NOT add a separate "AI Leads" page or tab — they live in the main leads list
- Do NOT change the lead data model — use existing `source` field
- Keep badges subtle — they should inform, not distract. A small colored pill, not a large banner.
- Follow existing badge/pill patterns in the codebase (status badges from T6.6 as reference)

## Failure Handling

- If a lead has `source: 'ai_agent'` but no AI-specific notes (data got lost), still show the badge but skip the detail banner content
- If call log doesn't have a `handled_by` field, fall back to checking the lead's source + call direction
- If no AI leads exist yet (agent hasn't enabled the feature), all the badge/filter code should be harmlessly dormant
