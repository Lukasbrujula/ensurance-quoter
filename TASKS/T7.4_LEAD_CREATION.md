# T7.4: Lead Creation — Parse AI Conversation into CRM Lead

**Priority:** High — this is the payoff: AI calls become actionable leads  
**Estimate:** 45 min  
**Phase:** 7 (Telnyx AI Agent)  
**Depends on:** T7.3 (webhook endpoint receives data)

---

## Context

After the AI webhook receives caller data (T7.3), we need to create a lead in the CRM. This connects the AI agent to the existing lead infrastructure built in Phases 1 and 6. The lead should appear in the agent's leads list with enough context for a productive callback.

## Tasks

### 1. Create lead from webhook data

In the webhook handler (or called from it), map the AI-collected data to a lead:

```typescript
const leadData = {
  agent_id: agentId,  // From agent lookup in T7.3
  name: webhookData.caller_name,
  phone: webhookData.callback_number,
  source: 'ai_agent',  // Key identifier for AI-collected leads
  status: 'new',
  notes: buildAINotes(webhookData),
  // Leave insurance-specific fields empty — agent fills these on callback
  // age, gender, state, tobacco, medical, coverage, term = null
};
```

### 2. Build rich notes from the AI conversation

The notes field should give the agent a concise callback brief:

```typescript
function buildAINotes(data: WebhookPayload, transcript?: TranscriptMessage[]): string {
  const lines: string[] = [];
  
  lines.push('AI Agent Call - ' + new Date().toLocaleString());
  lines.push('Caller type: ' + (data.caller_type || 'New inquiry'));
  lines.push('Reason: ' + data.reason);
  
  if (data.callback_preference) {
    lines.push('Callback preference: ' + data.callback_preference);
  }
  if (data.urgency === 'urgent') {
    lines.push('URGENT - caller indicated time-sensitive need');
  }
  
  return lines.join('\n');
}
```

### 3. Auto-schedule follow-up

Since the caller asked for a callback, auto-create a follow-up using the Phase 6 follow-up infrastructure:

```typescript
// Default: follow up within 1 hour (or use callback_preference to set time)
const followUpDate = parseCallbackPreference(data.callback_preference);
// "today" -> 1 hour from now
// "tomorrow morning" -> tomorrow at 9:00 AM
// "anytime" -> 1 hour from now
// Unparseable -> 1 hour from now (safe default)

await updateLeadFields(leadId, {
  follow_up_date: followUpDate.toISOString(),
  follow_up_note: 'Callback requested - ' + data.reason,
});
```

### 4. Create call log entry

Create a call log for the AI conversation using the existing call_logs infrastructure:

```typescript
await createCallLog({
  lead_id: leadId,
  direction: 'inbound',
  duration: conversation?.duration_seconds || null,
  transcript_text: formatTranscript(transcript),
  ai_summary: 'AI agent collected: ' + data.caller_name + ', calling about ' + data.reason + '. Callback requested: ' + (data.callback_preference || 'ASAP') + '.',
  coaching_hints: null, // No coaching for AI calls
});
```

### 5. Create activity log entry

Using Phase 6 activity timeline infrastructure:

```typescript
await logActivity({
  lead_id: leadId,
  agent_id: agentId,
  activity_type: 'call',
  title: 'AI agent handled inbound call',
  details: {
    direction: 'inbound',
    handled_by: 'ai_agent',
    caller_name: data.caller_name,
    reason: data.reason,
    urgency: data.urgency || 'normal',
    duration_seconds: conversation?.duration_seconds,
    has_transcript: !!transcript,
  },
});
```

### 6. Deduplicate: check if lead already exists

Before creating a new lead, check if a lead with the same phone number already exists for this agent:

```typescript
const existingLead = await findLeadByPhone(agentId, webhookData.callback_number);

if (existingLead) {
  // Update existing lead with new notes, call log, activity
  // Don't create a duplicate
  // Append to notes: "Additional AI call on [date]: [reason]"
  // Still create the call log and activity entries
} else {
  // Create new lead
}
```

This is important because the same person might call multiple times before the agent calls back.

## Success Criteria

- [ ] New lead is created in the leads table with source ai_agent
- [ ] Lead has status new and populated name, phone, notes
- [ ] Notes contain structured AI call summary (caller type, reason, urgency, callback preference)
- [ ] Follow-up is auto-scheduled based on callback preference
- [ ] Call log entry is created with transcript (if available)
- [ ] Activity log entry is created with handled_by ai_agent detail
- [ ] Duplicate check: if phone number matches existing lead, updates instead of creating duplicate
- [ ] Agent sees the lead in their /leads list immediately after the AI call
- [ ] npx tsc --noEmit passes

## Guardrails

- Do NOT pre-fill insurance-specific fields (age, coverage, term, tobacco, medical). The AI agent does not collect these.
- Do NOT create leads without at minimum a name and phone number. If webhook is missing these, log error and skip.
- Do NOT delete or overwrite existing lead data when deduplicating. Only append.
- Use the service role client for lead creation (this is a server action triggered by webhook, not a user session).
