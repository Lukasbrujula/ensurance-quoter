# T10.2: Back Navigation Arrows on All Pages

**Priority:** Medium — consistency and workflow continuity
**Estimate:** 30 min
**Phase:** 10 (Dashboard + UX Polish)

---

## 9 Pillars

### 1. Objective
Add a back-to-quoter navigation arrow on every page except the quote page itself. Currently only the settings page has this. The quoter is the primary workspace — agents should always be one click away from it.

### 2. Scope
- Add back arrow to: `/leads`, `/leads/[id]`, `/agents`, `/agents/[id]`, `/settings/*`, `/dashboard` (new)
- Do NOT add to: `/quote` (already there), `/auth/*` (pre-login)
- Arrow should be consistent in position and style across all pages

### 3. Dependencies
Find the existing back arrow implementation:
```bash
grep -rn "ArrowLeft\|ChevronLeft\|back\|Back" --include="*.tsx" app/settings/ components/settings/
grep -rn "useRouter\|router.back\|router.push" --include="*.tsx" app/ components/
```
Check how the settings page implements it to match the pattern.

### 4. Implementation

**Create a reusable BackToQuoter component:**

```typescript
// components/navigation/back-to-quoter.tsx
'use client';

import { useRouter } from 'next/navigation';
import { ArrowLeft } from 'lucide-react';
import { Button } from '@/components/ui/button';

interface BackToQuoterProps {
  label?: string;  // "Back to Quotes" or "Back to Leads" etc.
  href?: string;    // Override destination (default: /quote)
}

export function BackToQuoter({ label = 'Back to Quotes', href = '/quote' }: BackToQuoterProps) {
  const router = useRouter();
  
  return (
    <Button
      variant="ghost"
      size="sm"
      onClick={() => router.push(href)}
      className="gap-2 text-muted-foreground hover:text-foreground"
    >
      <ArrowLeft className="h-4 w-4" />
      {label}
    </Button>
  );
}
```

**Add to each page header:**

For pages that already have a header/title section, add the component above or to the left of the title:

```typescript
// Example: /leads page
<div className="space-y-4">
  <BackToQuoter />
  <h1 className="text-2xl font-bold">Leads</h1>
  {/* rest of page */}
</div>
```

For detail pages with breadcrumbs, the back arrow should go to the parent list:
- `/leads/[id]` → back arrow goes to `/leads` (not `/quote`)
- `/agents/[id]` → back arrow goes to `/agents`
- `/settings/*` → back arrow goes to `/settings` or `/quote`

**Keyboard shortcut (stretch goal):**
`Escape` key navigates back to quoter from any page. Only if this pattern exists elsewhere in the codebase.

### 5. Success Criteria
- [ ] Every non-auth, non-quote page has a back navigation arrow
- [ ] Arrow is visually consistent across all pages (same component, same position)
- [ ] `/leads/[id]` goes back to `/leads`, not directly to `/quote`
- [ ] `/agents/[id]` goes back to `/agents`
- [ ] Top-level pages (`/leads`, `/agents`, `/settings`) go back to `/quote`
- [ ] Arrow uses the existing settings page pattern (or improves on it consistently)
- [ ] `npx tsc --noEmit` passes

### 6. Failure Handling
- If pages have different header layouts, adapt the component placement to fit each
- If the settings page uses a different back pattern, normalize everything to the new component

### 7. Guardrails
- Do NOT change the quote page layout
- Do NOT add browser-style back buttons (these should be deterministic navigation, not history-based)
- Do NOT modify the top nav bar for this (back arrows are in the page content area, not the nav)
- Keep the component simple and consistent

### 8. Human Checkpoint
- Navigate to each page, verify the back arrow exists and goes to the right place
- Arrow should feel natural, not intrusive
