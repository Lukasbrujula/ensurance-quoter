# T6.7: Follow-Up Scheduling

**Priority:** Medium — agents need to track when to call leads back  
**Estimate:** 1 hr  
**Phase:** 6 (Lead Data Expansion + CRM)  
**Depends on:** T6.6 (lead status workflow exists)

---

## Context

Insurance agents juggle dozens of leads at different pipeline stages. A common request: "I talked to John, he said call back Thursday at 2pm." Without follow-up tracking, agents rely on sticky notes or memory. This task adds a lightweight scheduling system — not a full calendar, just a date/time + note attached to each lead with visual indicators.

## Tasks

### 1. Follow-up fields in lead detail
- In the lead detail view, add a "Follow-Up" section (or inline widget):
  - **Follow-up Date/Time** — Date + time picker (shadcn Calendar for date, time select for hours)
  - **Follow-up Note** — Text input (max 1000 chars, e.g., "Call back about $500K term quote, wife concerned about premium")
  - **Clear Follow-up** — Button to remove date + note (set both to null)
- Position: Prominent — near the top of lead detail, below status. This is an action-oriented field, not a detail field.

### 2. Follow-up indicator on leads list
- Add a visual indicator on leads that have an upcoming follow-up:
  - **Overdue** (past date): Red clock icon or red dot + "Overdue" tooltip with the date
  - **Today**: Orange/amber clock icon + "Today at [time]" tooltip
  - **Upcoming** (future): Blue clock icon + date tooltip
  - **No follow-up**: No indicator
- Position: In or near the Name column (don't add a full column for this — it's an indicator, not data)

### 3. Sort/filter by follow-up
- Add "Has Follow-up" as a filter option (show only leads with scheduled follow-ups)
- Add "Follow-up Date" as a sort option (show overdue first, then today, then upcoming)
- This helps agents start their day by seeing "who do I need to call back?"

### 4. Quick-schedule from leads list
- In the leads list, add a quick action on hover or in a dropdown: "Schedule Follow-up"
- Opens a small popover (not a full modal) with just date/time picker + note
- Saves without navigating to lead detail

### 5. Follow-up after call
- After a Telnyx call ends (in the post-call summary flow), offer a "Schedule Follow-up?" prompt
- Pre-filled with tomorrow at the same time as the call (sensible default)
- Agent can adjust or dismiss

### 6. Persistence
- Save `follow_up_date` and `follow_up_note` to the leads table in Supabase
- When a follow-up is completed (agent calls or manually clears it), set both to null
- Status change to "contacted" could auto-prompt: "Clear existing follow-up?"

## Success Criteria

- [ ] Lead detail shows follow-up date/time picker and note field
- [ ] Follow-up saves to Supabase (follow_up_date + follow_up_note columns)
- [ ] Leads list shows color-coded follow-up indicators (overdue red, today amber, upcoming blue)
- [ ] Follow-up indicators show tooltip with date/time and note
- [ ] Leads can be filtered to show only those with follow-ups
- [ ] Leads can be sorted by follow-up date
- [ ] Quick-schedule popover works from leads list without navigating to detail
- [ ] Post-call follow-up prompt appears after Telnyx calls
- [ ] Clearing follow-up sets both fields to null
- [ ] `npx tsc --noEmit` passes

## Dependencies

- T6.6 completed (lead status and detail enhancements exist)
- Leads list component
- Lead detail component
- Lead store — Supabase update functions
- Post-call summary flow (find with: `grep -r "call-summary\|PostCall\|callEnd" --include="*.tsx" components/`)
- shadcn/ui: Calendar, Popover, Input, Button (all installed)

## Guardrails

- Do NOT build a full calendar/scheduling system — this is per-lead follow-up only
- Do NOT add notifications/reminders (no push notifications, no email reminders — that's a future feature)
- Do NOT modify the calling pipeline — only add a prompt after the existing post-call flow
- Do NOT create a new database table — use the existing leads table columns
- Keep the quick-schedule popover minimal — date, time, note, save. No extra fields.

## Failure Handling

- If the time picker component is complex, fall back to a simple time Select with 30-min intervals (9:00 AM, 9:30 AM, etc.)
- If the post-call prompt integration is too coupled to the calling flow, skip it for now — the manual scheduling from lead detail and leads list is the core value
- If follow-up indicators add too much visual noise to the leads table, make them toggleable via a "Show Follow-ups" toggle above the table
