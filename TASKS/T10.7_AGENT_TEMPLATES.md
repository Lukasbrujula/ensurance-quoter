# T10.7: AI Agent Templates — Pre-Built Personas

**Priority:** Medium — reduces friction when creating agents
**Estimate:** 1 hr
**Phase:** 10 (Dashboard + UX Polish)

## 9 Pillars

### 1. Objective
The AI agent creation flow is too bare — agents have to configure everything from scratch. Add pre-built templates that set up name, greeting, system prompt, and voice for common use cases. Agent picks a template, customizes if needed, and deploys.

### 2. Scope
- Template selection step in the create agent flow
- 3-4 pre-built templates for insurance use cases
- Template previews (greeting, description, use case)
- "Start from scratch" option (current behavior)
- Templates are code-defined, not database-stored (for now)

### 3. Dependencies
- Create agent dialog from Phase 8 T8.3 (find: `grep -rn "CreateAgent\|create.*agent\|AgentCreate" --include="*.tsx" components/ app/agents/`)
- AI agent config from Phase 7 T7.2 (find: `grep -rn "buildInsuranceAgentConfig\|ai-config\|ai-prompts" --include="*.ts" lib/telnyx/`)
- Existing prompt generator (find: `cat lib/telnyx/ai-prompts.ts`)

**Write the new prompt builders:**

Each template needs its own system prompt following voice AI principles (from Phase 7 reference):
- 60% shorter than text equivalents
- One question per turn
- No markdown or formatting
- Goal-based, explicit caller paths

```typescript
// After Hours prompt
function buildAfterHoursPrompt(agentName: string, businessName: string): string {
  return `You are a friendly after-hours assistant for ${agentName} at ${businessName}.

The office is currently closed. Your job is to take a message and schedule a callback for the next business day.

Collect:
1. The caller's name
2. A brief message or reason for calling
3. The best number to reach them
4. Whether morning or afternoon works better for a callback

Keep it brief. Callers after hours want to leave a message quickly, not have a long conversation.

Do not discuss insurance rates, medical information, or give any advice. Simply take the message.

End by confirming: their name, callback number, and that ${agentName} will call back on the next business day.`;
}
```

Write similar builders for `buildSchedulerPrompt` and `buildFAQPrompt`.

### 5. Success Criteria
- [ ] Create agent dialog shows template selection as first step
- [ ] 4 templates available: Insurance Intake, After Hours, Scheduler, FAQ Handler
- [ ] Selecting a template pre-fills name, greeting, voice, and system prompt
- [ ] "Start from scratch" option skips to empty form
- [ ] Template cards show icon, name, and description
- [ ] Each template has a unique system prompt following voice AI best practices
- [ ] Agent can customize any pre-filled field before creating
- [ ] Creating from template works end-to-end (creates Telnyx assistant)
- [ ] `npx tsc --noEmit` passes

### 6. Failure Handling
- If template prompt builder fails, fall back to generic insurance intake prompt
- If templates array is empty (code error), show "Start from scratch" as the only option

### 7. Guardrails
- Do NOT store templates in the database (they're code-level, change with releases)
- Do NOT modify the existing Insurance Intake prompt from Phase 7 — add new ones alongside it
- Do NOT make templates mandatory — "Start from scratch" is always an option
- Keep prompt text under 1500 words each (voice AI constraint)

### 8. Human Checkpoint
- Open create agent dialog, pick "After Hours" template
- Verify fields are pre-filled correctly
- Create the agent, make a test call, verify it behaves like an after-hours agent
