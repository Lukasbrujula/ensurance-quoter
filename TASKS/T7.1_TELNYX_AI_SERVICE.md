# T7.1: Telnyx AI Service — API Wrapper for Assistants

**Priority:** Critical — everything in Phase 7 depends on this  
**Estimate:** 1.5 hrs  
**Phase:** 7 (Telnyx AI Agent)

---

## Context

Ensurance already has Telnyx integration for WebRTC calling (outbound/inbound via `@telnyx/webrtc`). Phase 7 adds a different Telnyx product: **AI Assistants** — server-side voice agents that answer calls autonomously. This requires a new service layer that talks to the Telnyx AI API (`https://api.telnyx.com/v2/ai/assistants`).

## Reference Implementation

**Before writing any code, clone and read from the reference repo:**

```bash
gh repo clone Lukasbrujula/growthlywhatsapp
```

**Files to read:**
1. `TELNYX_WORKING_CONFIG.md` — Read FIRST. Exact working payload, required fields, gotchas.
2. `src/voice/services/telnyx.service.ts` — Lines 1-300 for all TypeScript interfaces, lines 305-510 for the API wrapper class (request method, create/update/get/delete assistant).

**How to adapt:**
- The reference is NestJS with `@Injectable()` and `ConfigService`. Ours is Next.js — use a simple module with `fetch()` calls.
- The TypeScript interfaces (`TelnyxAssistantCreateDto`, `TelnyxVoiceSettings`, `TelnyxTranscriptionSettings`, `TelnyxTool`, `TelnyxAssistant`, etc.) can be copied almost verbatim into a types file.
- The `request()` method pattern (with retry, rate limit handling, error parsing) is solid — adapt it.

## Tasks

### 1. Create Telnyx AI types
- File: `lib/telnyx/ai-types.ts`
- Copy and adapt these interfaces from the reference `telnyx.service.ts`:
  - `TelnyxVoiceSettings` — voice, speed, background audio
  - `TelnyxTranscriptionSettings` — model, language, end-of-turn settings
  - `TelnyxTelephonySettings` — web calls support, noise suppression
  - `TelnyxWidgetSettings` — theme, position (needed for web test calls)
  - `TelnyxTool` — webhook, transfer, hangup, handoff types
  - `TelnyxAssistantCreateDto` — full creation payload
  - `TelnyxAssistantUpdateDto` — update payload (all optional + promote_to_main)
  - `TelnyxAssistant` — response type from API
  - `TelnyxConversation` — call/conversation record
  - `TelnyxTranscriptMessage` — individual transcript messages
- Keep only what we need — skip the voice listing, phone number purchasing, and insight group interfaces (we won't use those)

### 2. Create Telnyx AI service
- File: `lib/telnyx/ai-service.ts`
- Simple module (not a class) exporting async functions:

```typescript
const TELNYX_AI_BASE = 'https://api.telnyx.com/v2/ai';

// Core request helper with retry + rate limiting
async function telnyxAIRequest<T>(method, path, body?): Promise<T>

// Assistant CRUD
export async function createAssistant(dto: TelnyxAssistantCreateDto): Promise<TelnyxAssistant>
export async function updateAssistant(id: string, dto: TelnyxAssistantUpdateDto): Promise<TelnyxAssistant>
export async function getAssistant(id: string): Promise<TelnyxAssistant>
export async function deleteAssistant(id: string): Promise<void>

// Conversations (for retrieving AI call results)
export async function getConversations(assistantId: string, params?: { page_size?: number }): Promise<TelnyxConversation[]>
export async function getTranscript(conversationId: string): Promise<TelnyxTranscriptMessage[]>
```

### 3. Environment variable
- Uses existing `TELNYX_API_KEY` — same key works for both WebRTC and AI APIs
- Validate the key exists at call time, not module load time (serverless-friendly)

### 4. Add database columns for assistant config
- The agent needs to store their Telnyx AI assistant ID so we can manage it
- Add to `agent_settings` table:

```sql
ALTER TABLE agent_settings ADD COLUMN IF NOT EXISTS telnyx_ai_assistant_id TEXT;
ALTER TABLE agent_settings ADD COLUMN IF NOT EXISTS telnyx_ai_enabled BOOLEAN DEFAULT false;
```

- Update the TypeScript types for agent_settings to include these fields
