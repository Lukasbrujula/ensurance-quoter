# T8.4: /agents/[id] Detail Page — Config + Call History + Transcript Viewer

**Priority:** High — where agents review and manage individual AI agents
**Estimate:** 1.5 hrs
**Phase:** 8 (Agent Management + Transcripts + Usage)
**Depends on:** T8.3 (list page exists, navigation works)

---

## Context

When an agent clicks on an AI agent card from the list page, they land on the detail page. This is the command center for a single AI agent: edit its configuration, see its call history, read full conversation transcripts, and manage its status.

## 9 Pillars

### 1. Objective
Build `/agents/[id]` page with three sections: configuration panel, call history list, and conversation transcript viewer.

### 2. Scope
- New route: `app/agents/[id]/page.tsx`
- Agent config editing (name, greeting, voice, phone, status)
- Call history list from `ai_agent_calls`
- Transcript viewer pulling from `ai_transcripts`
- Delete agent with confirmation
- Back navigation to /agents

### 3. Dependencies
- T8.3 completed (list page and navigation exist)
- API routes from T8.2 (GET/PUT/DELETE /api/agents/[id], GET transcripts, GET usage)
- Existing page layout patterns (check `/leads/[id]` or `/settings` for reference)
- shadcn/ui: Tabs, Card, Input, Select, Textarea, Button, Badge, Dialog, ScrollArea, Skeleton

### 4. Implementation

**Page layout — two approaches (pick based on existing patterns):**

**Option A: Tabs layout**
```
/agents/[id]
+------------------------------------------+
| [Back] Agent Name            [Delete]     |
| Status: Active                            |
+------------------------------------------+
| [Configuration] [Call History] [Settings] |
+------------------------------------------+
| (tab content)                             |
+------------------------------------------+
```

**Option B: Scrollable sections**
```
/agents/[id]
+------------------------------------------+
| [Back] Agent Name            [Delete]     |
+------------------------------------------+
| Configuration                    [Save]   |
| (config form)                             |
+------------------------------------------+
| Recent Calls                 [View All]   |
| (call list)                               |
+------------------------------------------+
```

Choose based on what the codebase already does for detail pages (leads detail, settings).

**Configuration section:**
- Form fields (React Hook Form + Zod, matching existing patterns):
  - Name (text input)
  - Description (textarea, optional)
  - Phone Number (text input, E.164 format)
  - Greeting (textarea — what the AI says first)
  - Voice (select dropdown — list of Telnyx voices)
  - Status toggle (Active / Inactive switch)
- Save button with dirty state tracking
- On save: PUT /api/agents/[id], show toast
- Changes to greeting/voice/name sync to Telnyx assistant

**Call history section:**
- List of calls handled by this AI agent
- Each row shows:
  - Caller name (from ai_agent_calls.caller_name or linked lead name)
  - Phone number
  - Date/time (relative: "2 hours ago")
  - Duration (formatted: "1:42")
  - Status: "Lead created" (green) or "Processing" (yellow) or "Error" (red)
  - Expand arrow to show transcript inline
- Sorted by most recent first
- Pagination: show 10 at a time with "Load more" button
- Empty state: "No calls yet. Once your agent is active, calls will appear here."

**Transcript viewer:**
- When a call row is expanded (or clicked), show the full transcript
- Chat-style layout:
  - AI messages on the left (with bot icon)
  - Caller messages on the right (with user icon)
  - Timestamps on each message
  - System messages (if any) as centered gray text
- Pull from: `GET /api/agents/[id]/transcripts/[callId]`
- Loading skeleton while fetching
- If no transcript stored (Phase 7 calls without transcript storage), show "Transcript not available for this call"

**Delete agent:**
- Delete button in the page header (top right, destructive variant)
- Confirmation dialog: "Delete [Agent Name]? This will remove the agent and its Telnyx configuration. Call history will be preserved."
- On confirm: DELETE /api/agents/[id], redirect to /agents, show toast
- Loading state on confirm button

**Back navigation:**
- Back arrow or breadcrumb at top: "Agents / [Agent Name]"
- Links back to /agents

### 5. Success Criteria
- [ ] `/agents/[id]` page loads with agent data
- [ ] Configuration form pre-fills with current agent values
- [ ] Saving config updates both DB and Telnyx assistant
- [ ] Dirty state tracking prevents accidental navigation away
- [ ] Call history shows list of AI-handled calls with correct data
- [ ] Expanding a call row shows full transcript in chat-style layout
- [ ] Transcript messages are ordered correctly with role indicators
- [ ] Delete agent works with confirmation dialog
- [ ] After delete, user is redirected to /agents list
- [ ] Back navigation works
- [ ] Loading and error states handled
- [ ] Empty states for no calls, no transcripts
- [ ] `npx tsc --noEmit` passes

### 6. Failure Handling
- If agent not found (404), show "Agent not found" with link back to /agents
- If agent doesn't belong to user (403), redirect to /agents
- If save fails, show error toast, keep form data, revert dirty state
- If transcript fetch fails, show "Could not load transcript" inline (don't break the call history view)
- If delete fails, close dialog, show error toast

### 7. Guardrails
- Do NOT modify the leads detail page or quote workspace
- Do NOT build a full prompt editor (just a textarea for the greeting for now)
- Do NOT allow editing the model field from the UI (hardcoded to Qwen for safety)
- Do NOT show the raw system_prompt in the UI (it's auto-generated; show greeting only)
- Follow existing detail page patterns for layout, form handling, and navigation

### 8. Human Checkpoint
- Visual: does the detail page feel consistent with leads detail / settings?
- Config: can you edit name and greeting, save, and see it reflected?
- Calls: does the call history populate after an AI call?
- Transcript: does the chat-style viewer render messages correctly?
- Delete: does the confirmation flow work cleanly?
