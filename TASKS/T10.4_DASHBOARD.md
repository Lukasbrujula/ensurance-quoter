# T10.4: Dashboard Page — Agent Home Base

**Priority:** High — first thing agents see after login
**Estimate:** 2 hrs
**Phase:** 10 (Dashboard + UX Polish)

---

## 9 Pillars

### 1. Objective
Build `/dashboard` as the agent's home screen. Shows at-a-glance stats, upcoming scheduled calls/follow-ups, recent activity, and commission summary. This becomes the default redirect after login (instead of `/quote`).

### 2. Scope
- New route: `app/dashboard/page.tsx` (may already exist as placeholder from earlier phases)
- Stats cards: total leads, calls this week, close rate, total commission
- Calendar widget: upcoming follow-ups and AI-scheduled callbacks
- Recent activity feed (last 10 activities across all leads)
- Quick actions: "New Quote", "Add Lead", "View Agents"
- Dashboard icon in top nav (far left, before Leads)
- Post-login redirect changed from `/quote` to `/dashboard`

### 3. Dependencies
- Activity logs from Phase 6 (find: `grep -rn "activity_log\|getActivity" --include="*.ts" lib/`)
- Follow-up data from Phase 6 (find: `grep -rn "follow_up\|followUp" --include="*.ts" lib/`)
- Lead stats (find: `grep -rn "getLeads\|leads" --include="*.ts" lib/supabase/`)
- Call logs (find: `grep -rn "getCallLog\|call_log" --include="*.ts" lib/supabase/`)
- Commission calculation (find: `grep -rn "commission" --include="*.ts" lib/engine/`)
- Existing dashboard route (find: `ls app/dashboard/`)
- Recharts (already in project for charts)
- shadcn/ui: Card, Calendar, Badge, Button, ScrollArea

**Stats cards:**
Each card shows a number + trend indicator (up/down arrow with percentage vs last period):

```typescript
interface StatCard {
  title: string;
  value: string | number;
  trend?: { direction: 'up' | 'down' | 'flat'; percentage: number };
  icon: LucideIcon;
}
```

Data sources:
- **Total Leads**: COUNT from leads table where agent_id = user
- **Calls This Week**: COUNT from call_logs where created_at > start of week
- **Close Rate**: leads with status 'issued' / leads with status 'quoted' or later (percentage)
- **Commission MTD**: SUM of estimated commissions from quotes this month

**Upcoming calls/follow-ups section:**
- Query leads with `follow_up_date` in the next 7 days
- Include AI-scheduled callbacks (leads with source 'ai_agent' and follow_up_date set)
- Group by day: Today, Tomorrow, This Week, Next Week
- Each item shows: time, lead name, type (callback/follow-up), source (manual/AI)
- Click to navigate to lead detail
- "Add reminder" button to manually create a follow-up

**Calendar view (stretch goal):**
If Recharts or a calendar component can render a mini month view with dots on days that have events, include it. Otherwise, the grouped list above is sufficient.

**Recent activity feed:**
- Pull from activity_logs table, last 10 entries
- Show: icon, title, lead name, relative time ("2 hours ago")
- Click to navigate to the relevant lead
- Types: call, quote, status_change, enrichment, ai_agent_call, follow_up

**Quick actions:**
Top-right of the page:
- "New Quote" button → navigates to `/quote`
- "Add Lead" button → opens the add lead dialog (same one from leads page)

**Navigation update:**
- Add Dashboard icon to top nav, positioned LEFT of Leads
- Use `LayoutDashboard` or `Home` icon from Lucide
- Active state when on `/dashboard`

**Post-login redirect:**
Update the auth middleware or login redirect to go to `/dashboard` instead of `/quote`:
```bash
grep -rn "redirect\|/quote\|/leads" --include="*.ts" middleware.ts app/auth/ components/auth/
```

**API endpoint for dashboard stats:**
Create `GET /api/dashboard/stats` that returns aggregated data in a single call:
```typescript
// Returns:
{
  leads: { total: number, thisWeek: number, thisMonth: number },
  calls: { thisWeek: number, thisMonth: number, avgDuration: number },
  closeRate: number, // percentage
  commission: { mtd: number, ytd: number },
  upcomingFollowUps: Array<{ leadId, leadName, followUpDate, followUpNote, source }>,
  recentActivity: Array<{ id, type, title, leadName, createdAt }>,
}
```
Auth-guarded, uses auth client (not service role).

### 5. Success Criteria
- [ ] `/dashboard` page renders with all sections
- [ ] Stats cards show real data from the database
- [ ] Upcoming calls/follow-ups shows next 7 days of scheduled items
- [ ] AI-scheduled callbacks appear alongside manual follow-ups
- [ ] Recent activity feed shows last 10 activities with correct icons and timestamps
- [ ] Quick action buttons work (New Quote, Add Lead)
- [ ] Dashboard icon appears in top nav, left of Leads
- [ ] Post-login redirects to `/dashboard` instead of `/quote`
- [ ] Responsive: 4-col cards on desktop, 2-col tablet, 1-col mobile
- [ ] Page loads quickly (single API call for all stats)
- [ ] `npx tsc --noEmit` passes

### 6. Failure Handling
- If stats API fails, show skeleton cards with "Unable to load" message and retry button
- If no leads/calls exist yet (new agent), show friendly empty states: "No leads yet — start by getting a quote"
- If follow-up query returns empty, show "No upcoming calls — you're all caught up"

### 7. Guardrails
- Do NOT build a complex charting dashboard — keep it simple and fast
- Do NOT make the stats API slow (use efficient queries, consider materialized views later)
- Do NOT remove the ability to navigate to `/quote` directly (it's still accessible via nav)
- Follow existing page patterns for layout and data fetching
- Match the design language of the rest of the app

### 8. Human Checkpoint
- After login, does it redirect to dashboard?
- Do the stats numbers make sense given the data in the system?
- Do upcoming follow-ups match what's in the leads?
- Does the activity feed refresh when you perform actions on other pages?
